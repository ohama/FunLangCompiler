**MlirIR.fs** — Add three new cases to the `MlirOp` DU, after `ArithDivSIOp` and before `ReturnOp`:

```fsharp
| ArithCmpIOp  of result: MlirValue * predicate: string * lhs: MlirValue * rhs: MlirValue
| CfCondBrOp   of cond: MlirValue * trueLabel: string * trueArgs: MlirValue list
                                   * falseLabel: string * falseArgs: MlirValue list
| CfBrOp       of label: string * args: MlirValue list
```

`ArithCmpIOp` result is always I1. `CfCondBrOp` and `CfBrOp` are terminators (no result value).

**Printer.fs** — Add three new cases to `printOp` match, before the `ReturnOp` cases:

1. `ArithCmpIOp(result, predicate, lhs, rhs)` emits:
   `%result.Name = arith.cmpi %predicate, %lhs.Name, %rhs.Name : %printType(lhs.Type)`
   CRITICAL: type suffix is the OPERAND type (i64), NOT result type (i1).

2. `CfCondBrOp(cond, trueLabel, trueArgs, falseLabel, falseArgs)` emits:
   `cf.cond_br %cond.Name, ^trueLabel(trueArgs...), ^falseLabel(falseArgs...)`
   where args format is `(%v.Name : %printType(v.Type), ...)`. If args is empty, no parens.

3. `CfBrOp(label, args)` emits:
   `cf.br ^label(args...)`
   Same args format as above.

**Elaboration.fs** — Three changes:

(a) Add a private helper `elaborateCmp`:
```fsharp
let private elaborateCmp (env: ElabEnv) (predicate: string) (lhs: Expr) (rhs: Expr)
    : MlirValue * MlirOp list =
    let (lv, lops) = elaborateExpr env lhs
    let (rv, rops) = elaborateExpr env rhs
    let result = { Name = freshName env; Type = I1 }
    (result, lops @ rops @ [ArithCmpIOp(result, predicate, lv, rv)])
```

(b) Add cases to `elaborateExpr` match, before the catch-all `| _ ->`:
```fsharp
| Bool (b, _) ->
    let v = { Name = freshName env; Type = I1 }
    let n = if b then 1L else 0L
    (v, [ArithConstantOp(v, n)])
| Equal (lhs, rhs, _) -> elaborateCmp env "eq" lhs rhs
| NotEqual (lhs, rhs, _) -> elaborateCmp env "ne" lhs rhs
| LessThan (lhs, rhs, _) -> elaborateCmp env "slt" lhs rhs
| GreaterThan (lhs, rhs, _) -> elaborateCmp env "sgt" lhs rhs
| LessEqual (lhs, rhs, _) -> elaborateCmp env "sle" lhs rhs
| GreaterEqual (lhs, rhs, _) -> elaborateCmp env "sge" lhs rhs
```

(c) In `elaborateModule`, change the hardcoded `ReturnType = Some I64` to `ReturnType = Some resultVal.Type` so that bool-valued programs emit `-> i1` instead of `-> i64`.

Do NOT add `If`, `And`, or `Or` handling yet — those are Plan 03-02.
