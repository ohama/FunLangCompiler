**MlirIR.fs** — Add one new case to MlirOp DU:

```fsharp
| DirectCallOp of result: MlirValue * callee: string * args: MlirValue list
```

Add it after `CfBrOp` and before `ReturnOp`. The `callee` is the `@name` string (with `@` sigil). The `result.Type` encodes the return type.

**Printer.fs** — Add printOp case for DirectCallOp (before the ReturnOp cases):

```fsharp
| DirectCallOp(result, callee, args) ->
    let argNames = args |> List.map (fun (v: MlirValue) -> v.Name) |> String.concat ", "
    let argTypes = args |> List.map (fun (v: MlirValue) -> printType v.Type) |> String.concat ", "
    sprintf "%s%s = func.call %s(%s) : (%s) -> %s"
        indent result.Name callee argNames argTypes (printType result.Type)
```

Note: Use explicit `(v: MlirValue)` type annotation in lambdas to avoid F# ambiguity with `FuncOp.Name` (same pattern used in Phase 3 for CfCondBrOp/CfBrOp).

**Elaboration.fs** — Three changes:

1. **Extend ElabEnv** with `KnownFuncs` and `Funcs` fields:

```fsharp
type FuncSignature = {
    MlirName:   string        // e.g. "@fact" (with @ sigil)
    ParamTypes: MlirType list
    ReturnType: MlirType
}

type ElabEnv = {
    Vars:         Map<string, MlirValue>
    Counter:      int ref
    LabelCounter: int ref
    Blocks:       MlirBlock list ref
    KnownFuncs:   Map<string, FuncSignature>
    Funcs:        FuncOp list ref
}

let emptyEnv () : ElabEnv =
    { Vars = Map.empty; Counter = ref 0; LabelCounter = ref 0; Blocks = ref []
      KnownFuncs = Map.empty; Funcs = ref [] }
```

2. **Add LetRec case** in `elaborateExpr` (before the `| _ ->` fallthrough):

When `LetRec(name, param, body, inExpr, _)` is encountered:
- Build a **fresh** ElabEnv for the function body with `Vars = Map.ofList [(param, { Name = "%arg0"; Type = I64 })]`, fresh Counter/LabelCounter/Blocks, but **share** the parent's `Funcs` ref.
- Forward-declare `name` in the body env's `KnownFuncs` so recursive calls work.
- Elaborate the body with the fresh env.
- Assemble MlirRegion from the body env's blocks using the same multi-block logic as `elaborateModule` (if sideBlocks is empty, single block with ReturnOp; otherwise entry block + side blocks with ReturnOp patched onto last block).
- Create FuncOp with `Name = "@" + name`, `InputTypes = [I64]`, `ReturnType = Some bodyVal.Type`, `Body = { Blocks = allBodyBlocks }`.
- Append FuncOp to `env.Funcs`.
- Add `name` to the **parent** env's KnownFuncs.
- Elaborate `inExpr` with updated parent env.
- Return result of `inExpr`.

CRITICAL: The body env must have `Blocks = ref []` (fresh), NOT reuse `env.Blocks`. Only `Funcs` is shared. The body env's `Vars` contains ONLY the parameter — no outer variables (enforces "no free variables" constraint for Phase 4).

3. **Add App case** in `elaborateExpr` (before the `| _ ->` fallthrough):

When `App(funcExpr, argExpr, _)` is encountered:
- Match `funcExpr` as `Var(name, _)` and check if `name` is in `env.KnownFuncs`.
- If found: elaborate `argExpr`, create result MlirValue with freshName and `sig.ReturnType`, return `(result, argOps @ [DirectCallOp(result, sig.MlirName, [argVal])])`.
- If not found: `failwithf "Elaboration: unsupported App (only direct calls to known functions supported in Phase 4)"`.

4. **Update `elaborateModule`** to collect accumulated FuncOps:

After building the @main FuncOp, change the final return from `{ Funcs = [mainFunc] }` to `{ Funcs = env.Funcs.Value @ [mainFunc] }`. This places helper functions before @main (conventional order; MLIR accepts any order).
