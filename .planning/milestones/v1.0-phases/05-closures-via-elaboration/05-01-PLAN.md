2. Add 7 new cases to `MlirOp` DU (after DirectCallOp, before ReturnOp):
   ```
   | LlvmAllocaOp     of result: MlirValue * count: MlirValue * numCaptures: int
   | LlvmStoreOp      of value: MlirValue * ptr: MlirValue
   | LlvmLoadOp       of result: MlirValue * ptr: MlirValue
   | LlvmAddressOfOp  of result: MlirValue * fnName: string
   | LlvmGEPLinearOp  of result: MlirValue * ptr: MlirValue * index: int
   | LlvmReturnOp     of operands: MlirValue list
   | IndirectCallOp   of result: MlirValue * fnPtr: MlirValue * envPtr: MlirValue * arg: MlirValue
   ```

3. Add `IsLlvmFunc: bool` field to `FuncOp` record (after Body field).

4. Update `return42Module` — add `IsLlvmFunc = false` to the FuncOp record literal.

**Printer.fs changes:**

1. Add `| Ptr -> "!llvm.ptr"` to `printType`.

2. Add 7 new cases to `printOp` (see 05-RESEARCH.md Pattern 8 for exact format strings):
   - `LlvmAllocaOp(result, count, numCaptures)` → `%result = llvm.alloca %count x !llvm.struct<(ptr{, i64}^N)> : (i64) -> !llvm.ptr`
     - Build fields string: if numCaptures = 0 then "ptr" else "ptr" + String.concat "" (List.replicate numCaptures ", i64")
   - `LlvmStoreOp(value, ptr)` → `llvm.store %value, %ptr : {valueType}, !llvm.ptr` (no result assignment, like ReturnOp)
   - `LlvmLoadOp(result, ptr)` → `%result = llvm.load %ptr : !llvm.ptr -> {result.Type}`
   - `LlvmAddressOfOp(result, fnName)` → `%result = llvm.mlir.addressof %fnName : !llvm.ptr`
     - NOTE: fnName already includes @ sigil
   - `LlvmGEPLinearOp(result, ptr, index)` → `%result = llvm.getelementptr %ptr[%d] : (!llvm.ptr) -> !llvm.ptr, i64`
   - `LlvmReturnOp []` → `llvm.return`; `LlvmReturnOp operands` → `llvm.return %val : type, ...`
   - `IndirectCallOp(result, fnPtr, envPtr, arg)` → `%result = llvm.call %fnPtr(%envPtr, %arg) : !llvm.ptr, (!llvm.ptr, %argType) -> %resultType`

3. Update `printFuncOp`:
   - Change `"func.func"` to `if func.IsLlvmFunc then "llvm.func" else "func.func"`
   - The rest of the function (params, return type, body) stays the same.

**Elaboration.fs changes:**

1. Add `ClosureInfo` type (before FuncSignature):
   ```fsharp
   type ClosureInfo = {
       InnerLambdaFn: string    // MLIR name of the llvm.func body, e.g. "@closure_fn_0"
       NumCaptures:   int       // number of captured variables
   }
   ```

2. Add `ClosureInfo: ClosureInfo option` field to `FuncSignature` record.

3. Update `emptyEnv` — no change needed (KnownFuncs starts empty).

4. Add a `ClosureCounter: int ref` to `ElabEnv` for generating unique closure function names. Initialize to `ref 0` in `emptyEnv`.

5. Add `freeVars` helper function (pure, recursive). See 05-RESEARCH.md Pattern 4 for the complete implementation. Place it before `elaborateExpr`.

6. Extend the `Let` case in `elaborateExpr` to handle `Let(name, Lambda(outerParam, Lambda(innerParam, innerBody)), inExpr)`:
   - **Before the existing Let handler**, add a new match case for this specific pattern.
   - Algorithm (from 05-RESEARCH.md Pattern 6):
     a. Compute `captures = freeVars (Set.ofList [outerParam]) (Lambda(innerParam, innerBody, span)) |> Set.toList |> List.sort`
        - NOTE: The free vars are computed on the inner Lambda relative to outerParam being bound. This finds variables captured from outside the `let name outerParam = ...` scope.
     b. Generate fresh closure function name: `let closureFnName = sprintf "@closure_fn_%d" env.ClosureCounter.Value` then increment counter.
     c. **Compile inner lambda body (llvm.func):**
        - Create fresh env with `Counter = ref 0`, `LabelCounter = ref 0`, `Blocks = ref []`, `KnownFuncs = env.KnownFuncs`, `Funcs = env.Funcs`, `ClosureCounter = env.ClosureCounter`
        - Vars: `%arg0` (Ptr type, env pointer) and `%arg1` (I64, innerParam)
        - For each capture at index i: emit `LlvmGEPLinearOp(gepResult, %arg0, i+1)` then `LlvmLoadOp(captureVal, gepResult)`, bind capture name to captureVal in Vars
        - Elaborate innerBody with these bindings
        - Emit `LlvmReturnOp [bodyVal]`
        - Assemble FuncOp: `{ Name = closureFnName; InputTypes = [Ptr; I64]; ReturnType = Some I64; Body = { Blocks = allBodyBlocks }; IsLlvmFunc = true }`
        - Handle side blocks same as LetRec: if sideBlocks empty, single block with entry ops + LlvmReturnOp; otherwise entry block + side blocks with LlvmReturnOp on last block
     d. **Compile closure-maker func.func:**
        - Name: `"@" + name`
        - InputTypes: `[I64; Ptr]` (outerParam value + pre-allocated env pointer)
        - Body:
          - `%arg0` = outerParam (I64), `%arg1` = env ptr (Ptr)
          - Emit `LlvmAddressOfOp(fnPtrVal, closureFnName)` — get address of the lambda body
          - Emit `LlvmStoreOp(fnPtrVal, %arg1)` — store fn_ptr at env[0]
          - For each capture at index i: look up capture name in `env.Vars` to get the caller's value, emit `LlvmGEPLinearOp(slot, %arg1, i+1)` + `LlvmStoreOp(captureValue, slot)`
            - IMPORTANT: The capture values come from the OUTER env, not the inner lambda env. But here in the closure-maker, `%arg0` IS the outerParam — so if outerParam is a capture, use `{ Name = "%arg0"; Type = I64 }`. For other captures, they must already be in env.Vars.
            - Actually, for the `let add_n n = fun x -> x + n` case: outerParam is `n`, captures of `fun x -> x + n` relative to `{n}` bound is just `{n}` — so the capture IS outerParam. The fill code stores `%arg0` (outerParam value) at env[1].
          - Emit `ReturnOp [%arg1]` — return the filled env pointer
          - ReturnType: `Some Ptr`, IsLlvmFunc: false
     e. Add both FuncOps to `env.Funcs`.
     f. Add to `env.KnownFuncs`: `name -> { MlirName = "@" + name; ParamTypes = [I64]; ReturnType = Ptr; ClosureInfo = Some { InnerLambdaFn = closureFnName; NumCaptures = List.length captures } }`
     g. Elaborate `inExpr` with updated env.

7. Update ALL existing FuncOp record construction sites to include `IsLlvmFunc = false`:
   - In `LetRec` case: the funcOp record literal
   - In `elaborateModule`: the mainFunc record literal

8. Update ALL existing FuncSignature construction sites to include `ClosureInfo = None`:
   - In `LetRec` case: the `sig_` binding

**CRITICAL anti-patterns to avoid:**
- Do NOT use `return` (ReturnOp) inside llvm.func bodies — use LlvmReturnOp
- Do NOT use `llvm.return` (LlvmReturnOp) inside func.func bodies — use ReturnOp
- The closure-maker func.func body uses `ReturnOp [%arg1]` to return the env pointer
- The lambda llvm.func body uses `LlvmReturnOp [bodyVal]` to return the computed result
