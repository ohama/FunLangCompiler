# Milestone v10.0: FunLexYacc 네이티브 컴파일 지원

**Status:** ✅ SHIPPED 2026-03-30
**Phases:** 36-42
**Total Plans:** 9

## Overview

v10.0 resolves the remaining blockers for FunLexYacc self-hosting: compiler bug fixes that affect real-world code, hashtable string key support, CLI argument access, format string output, and multi-file import. Bug fixes come first (they affect all subsequent work), then runtime extensions build incrementally, and multi-file import caps the milestone as the most complex compiler change.

### Phase 40: Multi-file Import
**Goal**: `open "file.fun"` imports another file's bindings into the current scope
**Depends on**: Phase 39 (all runtime features complete; this is a compiler-only change)

### Phase 42: If-Match Nested Empty Block Fix
**Goal**: `if ... then ... else match ...` and `if ... then match ... else ...` patterns compile to valid MLIR without empty entry blocks
**Depends on**: Phase 41 (independent bug fix, but numbered after for ordering)
**Requirements**: FIX-04
**Success Criteria**:
  1. `if n = 0 then [] else match xs with | [] -> [] | h :: t -> [h]` compiles and runs correctly
  2. `if n = 0 then match xs with | _ -> [] else [xs]` compiles and runs correctly
  3. `let rec take n = fun xs -> if n = 0 then [] else match xs with | [] -> [] | h :: t -> h :: take (n - 1) t` compiles and runs correctly
  4. All existing 200+ E2E tests still pass
**Plans**: 1 plan

## Milestone Summary

**Key Decisions:**
- FIX-02: blocksAfterBind - 1 index for outer merge block targeting
- LangHashtableStr tag=-2 for runtime dispatch
- sprintf: format-specific C wrappers with elaboration dispatch
- printfn: desugars to println(sprintf ...) at elaboration time
- expandImports: HashSet push/pop for diamond import handling
- OpenDecl: two-pass collectModuleMembers + alias LetDecls
- If-match: isBranchTerminator for nested branch block patching

**Issues Resolved:**
- FIX-01: for-in mutable capture segfault (resolved in Phase 36, was pre-existing from v9.0)
- FIX-02: sequential if empty block (resolved in Phase 36)
- FIX-03: Bool module I1 coercion (resolved in Phase 36)
- FIX-04: if-match nested empty block (resolved in Phase 42)
- RT-01/RT-02: Hashtable string key ABI (resolved in Phase 37)

**Issues Deferred:**
- FLY-01/02/03: FunLexYacc full integration verification (deferred to future milestone)
- Hashtable.fun not synced with FunLang (uses backend-specific createStr/keysStr)

**Technical Debt Incurred:**
- None significant
