Add implementations after the existing hashtable functions (after lang_hashtable_trygetvalue). Follow the existing integer-key pattern but with string-content hashing and comparison.

**Static helpers (not declared in header):**

1. `static uint64_t lang_ht_str_hash(LangString* key)` — FNV-1a over key->data[0..key->length-1]:
   ```
   uint64_t h = UINT64_C(14695981039346656037);
   for (int64_t i = 0; i < key->length; i++) {
       h ^= (uint8_t)key->data[i];
       h *= UINT64_C(1099511628211);
   }
   return h;
   ```

2. `static LangHashEntryStr* lang_ht_str_find(LangHashtableStr* ht, LangString* key)` — bucket lookup with content equality:
   ```
   uint64_t bucket = lang_ht_str_hash(key) % (uint64_t)ht->capacity;
   LangHashEntryStr* e = ht->buckets[bucket];
   while (e != NULL) {
       if (e->key->length == key->length && memcmp(e->key->data, key->data, (size_t)key->length) == 0)
           return e;
       e = e->next;
   }
   return NULL;
   ```

3. `static void lang_ht_str_rehash(LangHashtableStr* ht, int64_t newCapacity)` — same pattern as `lang_ht_rehash` but with LangHashEntryStr and lang_ht_str_hash.

**Public functions (9 total):**

4. `lang_hashtable_create_str()` — GC_malloc LangHashtableStr, set tag=-2, capacity=16, size=0, GC_malloc bucket array, zero all buckets.

5. `lang_hashtable_get_str(ht, key)` — call lang_ht_str_find, return e->val or 0 if not found.

6. `lang_hashtable_set_str(ht, key, val)` — rehash if size*4 > capacity*3, find existing entry (update val) or create new LangHashEntryStr (GC_malloc), set key/val/next, prepend to bucket, increment size.

7. `lang_hashtable_containsKey_str(ht, key)` — return lang_ht_str_find != NULL ? 1 : 0.

8. `lang_hashtable_remove_str(ht, key)` — walk bucket chain with prev pointer, unlink matching entry, decrement size.

9. `lang_hashtable_keys_str(ht)` — iterate all buckets and entries, build LangCons* list with head = (int64_t)(uintptr_t)e->key (Ptr-as-I64 pattern, same as lang_hashtable_keys). Return the cons list (or NULL for empty).

10. `lang_hashtable_trygetvalue_str(ht, key)` — same pattern as lang_hashtable_trygetvalue: GC_malloc a 2-slot int64_t array, slot[0] = found (0/1), slot[1] = value. Return pointer to array.

11. `lang_index_get_str(collection, key)` — cast collection to LangHashtableStr*, call lang_hashtable_get_str. (String index is always hashtable, never array.)

12. `lang_index_set_str(collection, key, value)` — cast collection to LangHashtableStr*, call lang_hashtable_set_str.

**Critical implementation notes:**
- All GC_malloc calls use Boehm GC (already included). No manual free needed.
- String comparison MUST use memcmp on data with length check — never pointer identity.
- keys_str stores LangString* as int64_t in LangCons.head using `(int64_t)(uintptr_t)ptr` — this is the existing pattern and is GC-safe (conservative scan).
- Include `<string.h>` if not already included (for memcmp) — check existing includes first.
