  <name>Task 1: Implement two-pass flattenDecls with OpenDecl support</name>
  <files>src/FunLangCompiler.Compiler/Elaboration.fs</files>
  <action>
  Modify Elaboration.fs to implement OpenDecl handling. Three changes needed:

  1. **Add `collectModuleMembers` function** (new, before `flattenDecls`):
     - Signature: `let private collectModuleMembers (decls: Ast.Decl list) : Map<string, string list>`
     - Recursively scan decls. For `ModuleDecl(name, innerDecls, _)`, recurse into innerDecls with `modName = name`.
     - For `LetDecl(name, _, _)` when `modName <> ""` and `name <> "_"`: collect `(modName, modName + "_" + name)`.
     - For `LetRecDecl(bindings, _)` when `modName <> ""`: collect each `(modName, modName + "_" + name)`.
     - Group by module name, return `Map<string, string list>`.
     - See research 41-RESEARCH.md Pattern 1 for exact code.

  2. **Modify `flattenDecls` signature and add OpenDecl case**:
     - Change signature from `flattenDecls (modName: string) (decls: Ast.Decl list)` to `flattenDecls (moduleMembers: Map<string, string list>) (modName: string) (decls: Ast.Decl list)`.
     - Pass `moduleMembers` through recursive calls to `flattenDecls`.
     - Add explicit case BEFORE the wildcard `| _ -> [d]`:
       ```
       | Ast.Decl.OpenDecl([openedMod], s) ->
           match Map.tryFind openedMod moduleMembers with
           | Some qualifiedNames ->
               qualifiedNames |> List.map (fun qualifiedName ->
                   let shortName = qualifiedName.Substring(openedMod.Length + 1)
                   Ast.Decl.LetDecl(shortName, Ast.Var(qualifiedName, s), s))
           | None -> []
       ```
     - For multi-segment OpenDecl paths (`OpenDecl(path, _)` where `path.Length > 1`), emit `[]` (no-op). Match LangThree behavior.

  3. **Update `extractMainExpr`** to call two-pass:
     - Before `flattenDecls`, add: `let moduleMembers = collectModuleMembers decls`
     - Change: `let flatDecls = flattenDecls "" decls` to `let flatDecls = flattenDecls moduleMembers "" decls`

  IMPORTANT:
  - Do NOT change the existing short-name alias logic in elaborateExpr Let/LetRec handlers. That mechanism must remain (it handles intra-module references).
  - Use `Ast.Var(qualifiedName, s)` as alias body, NOT `Ast.Lambda(...)`. The qualified name is already bound.
  - Operator names like `++`, `^^`, `<|>` must appear as-is in the alias (unsanitized). Sanitization only happens in Printer.fs.
