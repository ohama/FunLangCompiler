# Milestone v12.0: Error Message Accuracy

**Status:** ✅ SHIPPED 2026-04-01
**Phases:** 47-50
**Total Plans:** 4

## Overview

에러 메시지의 줄 번호 정확성, 파서 에러 위치 추가, 테스트 안정성, 비교 연산 unboxing 버그 수정. Prelude 별도 파싱으로 줄 번호를 유저 소스 기준으로 정확하게 만들고, 파서 에러에 토큰 위치를 포함하며, fslit의 CHECK-RE로 테스트를 Prelude 변경에 독립적으로 만들고, List.choose 등에서 발생하는 비교 람다 unboxing 버그를 수정한다.

## Milestone Summary

**Key Decisions:**
- Two-phase parsing: preludeDecls @ userDecls, userSpan 사용, "<prelude>" 파일명
- lastParsedPos mutable (try 블록 전 선언) → F# scoping rule
- fslit CHECK-RE는 라인별 적용 — CHECK-RE: 접두사 라인만 정규식
- Ordinal comparison(>, <, >=, <=)에 coerceToI64 추가, Equal/NotEqual unchanged

**Issues Resolved:**
- User code line numbers off by 173 (Prelude prepended as text)
- Parse errors had no file:line:col position
- Error tests fragile to Prelude line count changes
- List.choose/filter with comparison predicates crashed (arith.cmpi + !llvm.ptr type mismatch)

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- None
