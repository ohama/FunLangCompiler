  <action>
Modify the `parseProgram` function in Program.fs to track the last consumed token's position and include it in parse error messages. The change is minimal (4 lines added/changed):

1. Add `let mutable lastParsedPos = FSharp.Text.Lexing.Position.Empty` BEFORE the `try` block (after `let mutable idx = 0`). This MUST be before `try` because F# try-with scoping does NOT allow the `with` handler to access variables defined inside `try`.

2. Inside the tokenizer closure, add `lastParsedPos <- pt.StartPos` after `lb.EndPos <- pt.EndPos` to track the position of every token consumed by the parser.

3. In the `with firstEx ->` handler, BEFORE the parseExpr fallback, build the positioned message:
```fsharp
let posMsg =
    if lastParsedPos = FSharp.Text.Lexing.Position.Empty then
        firstEx.Message
    else
        sprintf "%s:%d:%d: parse error" lastParsedPos.FileName lastParsedPos.Line lastParsedPos.Column
```

4. Change the final fallback from `raise firstEx` to `failwith posMsg`.

Do NOT modify the `main` function's error handler -- the existing `msg.Contains("parse error")` check already produces `[Parse] msg`, so the position will automatically appear in output.

Do NOT modify anything in LangThree (Lexer.fsl, Parser.fsy, etc.).
