<objective>
Unify HashSet to use the same LSB dispatch hash/equality as Hashtable.
Currently HashSet has its own `lang_hs_hash` (raw murmurhash only) and the
compiler untags values before add/contains. After this change, HashSet stores
tagged values as-is and uses `lang_ht_hash`/`lang_ht_eq` for hash/equality.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: C runtime — HashSet uses unified hash/equality</name>
  <files>src/FunLangCompiler.Compiler/lang_runtime.c</files>
  <action>
1. Remove `lang_hs_hash` function — replace with `lang_ht_hash` (already exists from Phase 90)

2. Update `lang_hashset_add`:
   - Change `lang_hs_hash(key)` to `lang_ht_hash(key)` 
   - Change equality check `e->key == key` to `lang_ht_eq(e->key, key)`
   - Key is now stored as-is (tagged int or pointer) — no change needed to storage

3. Update `lang_hashset_contains`:
   - Change `lang_hs_hash(key)` to `lang_ht_hash(key)`
   - Change equality check `e->key == key` to `lang_ht_eq(e->key, key)`

4. Update `lang_for_in_hashset`:
   - Remove `LANG_TAG_INT(e->key)` — keys are now stored tagged, pass directly
   - Change to: `fn(closure, e->key);`
  </action>
  <verify>`dotnet build src/FunLangCompiler.Cli` succeeds</verify>
  <done>HashSet uses unified LSB dispatch hash/equality. lang_hs_hash removed.</done>
</task>
