Look at existing test files (e.g., `tests/compiler/66-09-int-hashtablestr-prelude.sh`) to understand the FsLit test format. Tests are shell scripts with embedded FunLang source and expected output markers.

**Test 93-01: Tuple as hashtable key**
Write a test that:
- Creates a hashtable
- Uses `hashtable_set ht (1, "a") 100`
- Uses `hashtable_set ht (2, "b") 200`
- Retrieves with `hashtable_get ht (1, "a")` — expects 100
- Retrieves with `hashtable_get ht (2, "b")` — expects 200
- Tests that a different tuple `(1, "b")` is NOT the same key (containsKey returns false or get fails)
- Tests that structurally equal tuple `(1, "a")` (constructed separately) retrieves the same value

**Test 93-02: Record as hashtable key**
Write a test that:
- Defines a record type (or uses inline record syntax)
- Uses records as hashtable keys
- Verifies structural equality for records (same fields, same values = same key)

**Test 93-03: Generic structural equality**
Write a test that exercises the `=` operator on:
- Tuples: `(1, "a") = (1, "a")` — true
- Tuples: `(1, "a") = (1, "b")` — false
- Lists: `[1, 2, 3] = [1, 2, 3]` — true
- Lists: `[1, 2] = [1, 2, 3]` — false
- ADTs: define a simple type like `type Color = Red | Green | Blue` and test `Red = Red` (true), `Red = Green` (false)
- ADTs with data: `type Opt = None | Some of int`, test `Some 1 = Some 1` (true), `Some 1 = Some 2` (false)
- Nested: `(1, [2, 3]) = (1, [2, 3])` — true

**Note on = operator:** Check how equality currently works in the compiler. If `=` on heap types currently compares pointers (not structural), the generic equality change in lang_ht_eq won't automatically affect `=`. Check if `=` for non-int types is compiled to a C call or LLVM icmp. If it's `icmp`, it's pointer equality and we may need to update the compiler to call `lang_ht_eq` for structural equality. If this is the case, update Elaboration.fs to emit a call to a new `@lang_generic_eq` extern for pointer comparisons instead of `icmp`.

If `=` already works structurally via C calls, just write the tests. If not, add the compiler change to emit `lang_generic_eq` calls and declare the function in lang_runtime.h (remove `static` from lang_ht_eq or create a non-static wrapper `lang_generic_eq` that calls `lang_ht_eq`).
