# Milestone v14.0: FunLang Standard Library Sync

**Status:** 🚧 In Progress
**Phases:** 54-56
**Total Plans:** TBD

## Overview

FunLang v13.0 이후 Prelude/String.fun과 Prelude/List.fun에 추가된 함수들을 컴파일러에 반영한다. String 모듈 7개 함수 중 5개는 C 런타임 신규 구현과 Elaboration 디스패치가 필요하고, 2개는 기존 런타임 함수의 별칭이다. List 모듈은 17개 순수 FunLang 함수를 Prelude 파일에 복사하는 것으로 충분하다. 모든 신규 함수에 대해 E2E 테스트로 검증한다.

## Phases

- [ ] **Phase 54: String Runtime & Elaboration** - 5개 신규 C 런타임 함수 구현 + Elaboration 디스패치 + Prelude/String.fun 동기화
- [ ] **Phase 55: List Prelude Sync** - Prelude/List.fun에 17개 신규 함수 동기화
- [ ] **Phase 56: E2E Testing** - String 7개 + List 17개 신규 함수 E2E 테스트

## Phase Details

### Phase 54: String Runtime & Elaboration
**Goal**: FunLang 프로그램에서 String.split, String.indexOf, String.replace, String.toUpper, String.toLower, String.join, String.substring을 사용할 수 있음
**Depends on**: Nothing (first phase of v14.0)
**Requirements**: STR-01, STR-02, STR-03, STR-04, STR-05, STR-06, STR-07, RT-01, RT-02, RT-03, RT-04, RT-05, RT-06
**Success Criteria** (what must be TRUE):
  1. `String.split "a,b,c" ","` compiles and returns a list `["a"; "b"; "c"]`
  2. `String.indexOf "hello" "ll"` compiles and returns the integer index (2)
  3. `String.replace "foo bar" "bar" "baz"` compiles and returns `"foo baz"`
  4. `String.toUpper "hello"` and `String.toLower "HELLO"` compile and return correct cased strings
  5. `String.join "," ["a"; "b"; "c"]` and `String.substring "hello" 1 3` compile via existing runtime aliases

- [ ] 54-02: Elaboration dispatch + Prelude/String.fun sync (5 new builtins + join/substring aliases)

### Phase 55: List Prelude Sync
**Goal**: FunLang 프로그램에서 List.init, List.find, List.findIndex, List.partition, List.groupBy, List.scan, List.replicate, List.collect, List.pairwise, List.sumBy, List.sum, List.minBy, List.maxBy, List.contains, List.unzip, List.forall, List.iter을 사용할 수 있음
**Depends on**: Nothing (independent of Phase 54)
**Requirements**: LIST-01
**Success Criteria** (what must be TRUE):
  1. `List.replicate 3 "x"` compiles and returns `["x"; "x"; "x"]`
  2. `List.find (fun x -> x > 2) [1; 2; 3; 4]` compiles and returns `3`
  3. `List.partition (fun x -> x > 2) [1; 2; 3; 4]` compiles and returns `([3; 4], [1; 2])`
  4. `List.forall (fun x -> x > 0) [1; 2; 3]` compiles and returns `true`
  5. All 17 new List functions are present and correct in Prelude/List.fun
