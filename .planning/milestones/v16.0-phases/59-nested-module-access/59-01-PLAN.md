Make these 5 coordinated changes to Elaboration.fs. Read the file first to find exact line numbers. The RESEARCH.md has complete code examples for each change.

**Change 1 — collectModuleMembers (NEST-02):**
Rewrite the `scan` inner function to accept two parameters: `dotPath` (for map key, e.g., "Outer.Inner") and `underPath` (for member name prefix, e.g., "Outer_Inner"). Currently scan takes a single `modName`. Change the signature to `scan (dotPath: string) (underPath: string) (ds: Ast.Decl list)`. In the ModuleDecl arm, compute `childDot = if dotPath = "" then name else dotPath + "." + name` and `childUnder = if underPath = "" then name else underPath + "_" + name`. In LetDecl/LetRecDecl arms, use `underPath` for the qualified name and `dotPath` as the map key. Change the initial call from `scan "" decls` to `scan "" "" decls`. See RESEARCH.md "Complete collectModuleMembers Rewrite" for exact code.

**Change 2 — flattenDecls ModuleDecl arm (NEST-01):**
In the `flattenDecls` function's `ModuleDecl` arm, change `flattenDecls moduleMembers name innerDecls` to compute `let childPrefix = if modName = "" then name else modName + "_" + name` and call `flattenDecls moduleMembers childPrefix innerDecls`. This ensures nested modules get the full underscore-joined prefix. See RESEARCH.md "Complete flattenDecls Rewrite".

**Change 3 — flattenDecls OpenDecl arm (NEST-04):**
In the `flattenDecls` function's `OpenDecl` arm, change the map lookup from using `List.last` (or whatever single-segment extraction exists) to `let openedKey = path |> String.concat "."`. Then for shortName extraction, compute `let underscorePrefix = openedKey.Replace(".", "_")` and use `qualifiedName.Substring(underscorePrefix.Length + 1)`. See RESEARCH.md "Complete OpenDecl arm".

**Change 4 — tryDecodeModulePath helper (NEST-03):**
Add a new private recursive helper function `tryDecodeModulePath` above the `elaborateExpr` function. It takes an `Expr` and returns `string list option`. Pattern: `Constructor(name, None, _) -> Some [name]`, `FieldAccess(inner, field, _) -> tryDecodeModulePath inner |> Option.map (fun segs -> segs @ [field])`, `_ -> None`. See RESEARCH.md "tryDecodeModulePath Helper".

**Change 5 — New elaboration arms (NEST-03):**
Add two new pattern match arms in `elaborateExpr`:

(a) Insert BEFORE the existing `App(FieldAccess(Constructor(...)))` arm (~line 2503):
```
| App(FieldAccess(FieldAccess(_, _, _) as innerExpr, memberName, fspan), argExpr, span)
    when (tryDecodeModulePath innerExpr).IsSome
      && not (Map.containsKey memberName env.TypeEnv) ->
```
Decode segments via tryDecodeModulePath, concat with "_", re-elaborate as `App(Var(varName, fspan), argExpr, span)`.

(b) Insert BEFORE the existing `FieldAccess(Constructor(...))` arm (~line 3232):
```
| FieldAccess(FieldAccess(_, _, _) as innerExpr, memberName, span)
    when (tryDecodeModulePath innerExpr).IsSome ->
```
Decode segments, check TypeEnv for constructor case, otherwise re-elaborate as `Var(allSegments |> String.concat "_", span)`.

CRITICAL: Do NOT modify the existing single-level arms (`FieldAccess(Constructor(...))` and `App(FieldAccess(Constructor(...)))`). The new arms must come BEFORE them and only match when innerExpr is itself a FieldAccess (2+ level nesting).
