  <action>
Create `src/FunLangCompiler.Compiler/ProjectFile.fs` with module name `ProjectFile`.

Define types:
```fsharp
type Target = { Name: string; Main: string }
type FunProjConfig = {
    Name: string option
    PreludePath: string option
    Executables: Target list
    Tests: Target list
    ProjectDir: string
}
```

Implement `parseFunProj (content: string) (projectDir: string) : FunProjConfig`:
- Line-by-line parser. Trim each line. Skip blank lines and lines starting with `#`.
- Track current section via a discriminated union or string: `"project"`, `"executable"`, `"test"`, or `"none"`.
- Section headers:
  - `[project]` -> switch to project section
  - `[[executable]]` -> push a new executable target accumulator, switch to executable section
  - `[[test]]` -> push a new test target accumulator, switch to test section
- Key-value parsing: match lines like `key = "value"` (strip quotes from value).
  - In project section: `name` -> set Name, `prelude` -> set PreludePath
  - In executable section: `name` -> set current target Name, `main` -> set current target Main
  - In test section: same as executable
- After all lines processed, collect accumulated targets into Executables and Tests lists.
- Return FunProjConfig with the parsed values and the given projectDir.

Implement `findFunProj () : string option`:
- Start from `System.IO.Directory.GetCurrentDirectory()`.
- Walk up directory tree checking for `funproj.toml` file existence.
- Return `Some fullPath` if found, `None` if reached filesystem root without finding it.

Implement `resolveTarget (config: FunProjConfig) (target: Target) : string`:
- `System.IO.Path.GetFullPath(System.IO.Path.Combine(config.ProjectDir, target.Main))`

Important: Keep the parser simple. No need to handle escape sequences in strings, multi-line strings, integers, booleans, nested tables, or inline tables. Only handle: section headers `[project]` / `[[executable]]` / `[[test]]`, key = "string_value" pairs, comments (#), blank lines.

Edge cases to handle:
- Whitespace around `=` sign: `name = "foo"` and `name="foo"` both work
- Lines with only whitespace are skipped
- Unknown keys within a known section are silently ignored
- Unknown sections are silently ignored (treat as "none")

**Revised approach -- create a simple shell-based test script instead:**

Create file `tests/compiler/60-01-funproj-parse.sh`:
```bash
#!/bin/bash
set -e
cd "$(dirname "$0")/../.."
# Build the compiler library
dotnet build src/FunLangCompiler.Compiler/FunLangCompiler.Compiler.fsproj -c Release --nologo -v q

# Run an FSI script that tests parseFunProj
dotnet fsi --reference:src/FunLangCompiler.Compiler/bin/Release/net10.0/FunLangCompiler.Compiler.dll << 'SCRIPT'
open ProjectFile

let toml = """
[project]
name = "demo"
prelude = "Prelude"

[[executable]]
name = "app"
main = "src/main.fun"

[[test]]
name = "unit"
main = "tests/unit.fun"

[[test]]
name = "integ"
main = "tests/integ.fun"
"""

let config = parseFunProj toml "/tmp/proj"
assert (config.Name = Some "demo")
assert (config.PreludePath = Some "Prelude")
assert (config.Executables.Length = 1)
assert (config.Executables.[0].Name = "app")
assert (config.Executables.[0].Main = "src/main.fun")
assert (config.Tests.Length = 2)
assert (config.Tests.[0].Name = "unit")
assert (config.Tests.[1].Name = "integ")
assert (config.ProjectDir = "/tmp/proj")

// Test resolveTarget
let resolved = resolveTarget config config.Executables.[0]
assert (resolved.EndsWith("src/main.fun"))

printfn "ALL TESTS PASSED"
SCRIPT
```

Make it executable: `chmod +x tests/compiler/60-01-funproj-parse.sh`
