# Requirements Archive: v2.0 Data Types & Pattern Matching

**Status:** SHIPPED
**Completed:** 2026-03-26
**Summary:** 20/20 shipped

## GC Runtime

- [x] **GC-01**: Boehm GC 런타임 통합 — `GC_INIT()` + `GC_malloc` 외부 함수 선언, `-lgc` 링크
- [x] **GC-02**: v1 클로저 `llvm.alloca` → `GC_malloc` 마이그레이션 (힙 탈출 안전성)
- [x] **GC-03**: `print` / `println` 빌트인 — `printf` libc 호출로 stdout 출력

## String

- [x] **STR-01**: 문자열 리터럴 → 힙 할당 `{i64 length, ptr data}` 구조체
- [x] **STR-02**: 문자열 동등 비교 (`=`, `<>`) → `strcmp` libc 호출
- [x] **STR-03**: `string_length` 빌트인
- [x] **STR-04**: `string_concat` 빌트인 (+ 연산자 포함)
- [x] **STR-05**: `to_string` 빌트인 (int/bool → string)

## Tuple

- [x] **TUP-01**: 튜플 생성 → GC_malloc'd 구조체 (N개 포인터 필드)
- [x] **TUP-02**: `let (a, b) = ...` 튜플 디스트럭처링 → GEP + load
- [x] **TUP-03**: 튜플 패턴 매칭 (match에서 TuplePat)

## List

- [x] **LIST-01**: 빈 리스트 `[]` → null 포인터
- [x] **LIST-02**: cons `h :: t` → GC_malloc'd cons cell `{head: ptr, tail: ptr}`
- [x] **LIST-03**: 리스트 리터럴 `[e1; e2; ...]` → 중첩 cons로 디슈가
- [x] **LIST-04**: 리스트 패턴 매칭 (`[]` / `h :: t`) → null check + GEP

## Pattern Matching

- [x] **PAT-01**: `match` 식 → 결정 트리 기반 cf.cond_br 체인 컴파일
- [x] **PAT-02**: 상수 패턴 (int, bool) → `arith.cmpi eq` 비교
- [x] **PAT-03**: 문자열 상수 패턴 → `strcmp` 비교
- [x] **PAT-04**: 와일드카드/변수 패턴 → 무조건 매치 + 바인딩
- [x] **PAT-05**: 비소진 매치 런타임 에러 → `@lang_match_failure` 호출

## Out of Scope (deferred)

| Feature | Reason |
|---------|--------|
| Precise/moving GC | Boehm 보수적 GC로 충분, 정밀 GC는 수개월 작업 |
| 참조 카운팅 | 순환 참조 누수, Boehm이 투명하게 처리 |
| 언박싱 튜플 최적화 | 균일 박싱 표현으로 v2 단순성 유지, v3로 미룸 |
| 문자열 인터닝 | v2에서 모든 문자열 독립 할당, strcmp로 비교 |
| ADT/GADT 컴파일 | v3 범위 |
| 레코드 타입 | v3 범위 |
| 예외 처리 (raise/try-with) | setjmp/longjmp 복잡도, v3 범위 |
| or-패턴 (P1 \| P2) | 결정 트리 분기 공유 복잡도, v2에서 미지원 |
| when 가드 | 패턴 매칭 후 조건 분기, v2에서 미지원 |
| char 타입 컴파일 | FunLang에서 사용 빈도 낮음, v3 미룸 |
| tail call optimization | LLVM 최선-노력 TCO에 의존, 명시적 보장 안 함 |
