  <name>Task 1: Migrate closure alloca to GC_malloc and emit GC_init in @main</name>
  <files>src/FunLangCompiler.Compiler/Elaboration.fs</files>
  <action>
Three coordinated changes in Elaboration.fs:

**Change A -- Replace alloca with GC_malloc in App dispatch (closure-making call):**

Find the `Some sig_` branch in the `App` match (around line 345-357) where closure environment is allocated. Replace the two-op `ArithConstantOp(countVal, 1L) + LlvmAllocaOp(envPtrVal, countVal, ci.NumCaptures)` sequence with:

```fsharp
| Some sig_ ->
    let ci = sig_.ClosureInfo.Value
    let (argVal, argOps) = elaborateExpr env argExpr
    let bytesVal = { Name = freshName env; Type = I64 }
    let envPtrVal = { Name = freshName env; Type = Ptr }
    let resultVal = { Name = freshName env; Type = Ptr }
    let setupOps = [
        ArithConstantOp(bytesVal, int64 ((ci.NumCaptures + 1) * 8))
        LlvmCallOp(envPtrVal, "@GC_malloc", [bytesVal])
    ]
    let callOp = DirectCallOp(resultVal, sig_.MlirName, [argVal; envPtrVal])
    (resultVal, argOps @ setupOps @ [callOp])
```

The key differences:
- `countVal` (1L for alloca element count) becomes `bytesVal` (byte count = (numCaptures + 1) * 8)
- `LlvmAllocaOp` becomes `LlvmCallOp(envPtrVal, "@GC_malloc", [bytesVal])`
- The rest (DirectCallOp to the closure-maker) stays exactly the same

**Change B -- Prepend GC_init to @main entry block:**

In `elaborateModule`, AFTER building `allBlocks` but BEFORE constructing `mainFunc`, prepend `LlvmCallVoidOp("@GC_init", [])` to the first block's body:

```fsharp
let gcInitOp = LlvmCallVoidOp("@GC_init", [])
let allBlocksWithGC =
    match allBlocks with
    | [] -> allBlocks
    | entryBlock :: rest ->
        { entryBlock with Body = gcInitOp :: entryBlock.Body } :: rest
```

Then use `allBlocksWithGC` in `mainFunc` instead of `allBlocks`.

**Change C -- Populate Globals and ExternalFuncs in MlirModule:**

In `elaborateModule`, change the final MlirModule construction from:
```fsharp
{ Funcs = env.Funcs.Value @ [mainFunc] }
```
To:
```fsharp
let externalFuncs = [
    { ExtName = "@GC_init";   ExtParams = [];    ExtReturn = None;     IsVarArg = false }
    { ExtName = "@GC_malloc"; ExtParams = [I64]; ExtReturn = Some Ptr; IsVarArg = false }
]
{ Globals = []; ExternalFuncs = externalFuncs; Funcs = env.Funcs.Value @ [mainFunc] }
```

The `Globals` field is empty for now (populated in plan 07-03 when print/println is added).

CRITICAL: The symbol name MUST be `@GC_init` (lowercase `i`), NOT `@GC_INIT`. `GC_INIT` is a C preprocessor macro, not a linkable symbol. Using uppercase will produce "undefined symbol" linker error.

This test exercises: closure-maker call with GC_malloc'd env, escaped closure applied after maker returns, correct capture value access from heap.
