Four coordinated changes:

**Change A -- Add Globals fields to ElabEnv:**

Add two new fields to the `ElabEnv` record:
```fsharp
type ElabEnv = {
    Vars:           Map<string, MlirValue>
    Counter:        int ref
    LabelCounter:   int ref
    Blocks:         MlirBlock list ref
    KnownFuncs:     Map<string, FuncSignature>
    Funcs:          FuncOp list ref
    ClosureCounter: int ref
    Globals:        (string * string) list ref   // NEW: (name, rawValue) pairs
    GlobalCounter:  int ref                       // NEW: counter for unique global names
}
```

Update `emptyEnv` to initialize: `Globals = ref []; GlobalCounter = ref 0`.

**Change B -- Add `addStringGlobal` helper:**

Add a private helper that checks for content-based deduplication:
```fsharp
let private addStringGlobal (env: ElabEnv) (rawValue: string) : string =
    match env.Globals.Value |> List.tryFind (fun (_, v) -> v = rawValue) with
    | Some (name, _) -> name
    | None ->
        let idx = env.GlobalCounter.Value
        env.GlobalCounter.Value <- idx + 1
        let name = sprintf "@__str_%d" idx
        env.Globals.Value <- env.Globals.Value @ [(name, rawValue)]
        name
```

**Change C -- Add print/println special cases in elaborateExpr:**

Add these cases BEFORE the existing `| App (funcExpr, argExpr, _) ->` match arm. They must come before the general App case so they match first.

```fsharp
| App (Var ("print", _), String (s, _), _) ->
    let globalName = addStringGlobal env s
    let ptrVal  = { Name = freshName env; Type = Ptr }
    let fmtRes  = { Name = freshName env; Type = I32 }
    let unitVal = { Name = freshName env; Type = I64 }
    let ops = [
        LlvmAddressOfOp(ptrVal, globalName)
        LlvmCallOp(fmtRes, "@printf", [ptrVal])
        ArithConstantOp(unitVal, 0L)
    ]
    (unitVal, ops)

| App (Var ("println", _), String (s, _), _) ->
    let globalName = addStringGlobal env (s + "\n")
    let ptrVal  = { Name = freshName env; Type = Ptr }
    let fmtRes  = { Name = freshName env; Type = I32 }
    let unitVal = { Name = freshName env; Type = I64 }
    let ops = [
        LlvmAddressOfOp(ptrVal, globalName)
        LlvmCallOp(fmtRes, "@printf", [ptrVal])
        ArithConstantOp(unitVal, 0L)
    ]
    (unitVal, ops)
```

The `unitVal` (i64 0) represents the unit return value of print/println so it can participate in let-binding chains like `let _ = print "hello" in 42`.

NOTE: For Phase 7 scope, only handle the `String` literal argument case. Variable string arguments require Phase 8 string type support.

**Change D -- Transfer globals to MlirModule and add @printf to ExternalFuncs:**

In `elaborateModule`, update the final MlirModule construction:

```fsharp
let globals = env.Globals.Value |> List.map (fun (name, value) -> StringConstant(name, value))
let externalFuncs = [
    { ExtName = "@GC_init";   ExtParams = [];    ExtReturn = None;     IsVarArg = false }
    { ExtName = "@GC_malloc"; ExtParams = [I64]; ExtReturn = Some Ptr; IsVarArg = false }
    { ExtName = "@printf";    ExtParams = [Ptr]; ExtReturn = Some I32; IsVarArg = true  }
]
{ Globals = globals; ExternalFuncs = externalFuncs; Funcs = env.Funcs.Value @ [mainFunc] }
```

Always emit all three external declarations (GC_init, GC_malloc, printf). Unused declarations are harmless and simplify the logic. This replaces the two-declaration list from plan 07-02.

IMPORTANT: When elaborating inner closure bodies (the `innerEnv` in the Let/Lambda case), pass the SAME `Globals` and `GlobalCounter` refs so string globals from nested scopes are accumulated correctly:
```fsharp
let innerEnv : ElabEnv =
    { ... existing fields ...
      Globals = env.Globals
      GlobalCounter = env.GlobalCounter }
```
Do the same for `bodyEnv` in the `LetRec` case.
