<objective>
Move closure capture stores from the 2-lambda maker func.func to the call site, fixing the SSA scope violation that prevents 3+ arg curried functions from compiling (Issue #5).

Purpose: The maker body is a separate func.func and cannot reference SSA values from the outer elaboration scope. Moving capture stores to the call site (where SSA values are in scope) fixes this structurally. Then remove the 3+ lambda guard so all multi-lambda functions use the 2-lambda path.

Output: Modified Elaboration.fs with caller-side capture stores, working 3+ arg curried functions.
</objective>
