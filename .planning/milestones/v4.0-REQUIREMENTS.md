## v1 Requirements

### ADT Infrastructure
- [x] **ADT-01**: TypeDecl processing — constructor name -> tag index mapping (ElabEnv.TypeEnv)
- [x] **ADT-02**: ExceptionDecl processing — exception constructor -> tag mapping
- [x] **ADT-03**: MatchCompiler CtorTag extension — AdtCtor, RecordCtor + desugarPattern
- [x] **ADT-04**: elaborateProgram entry point — TypeDecl/RecordDecl/ExceptionDecl pre-processing

### ADT Construction & Pattern Matching
- [x] **ADT-05**: Nullary constructor compilation — 16-byte {tag, null} heap block
- [x] **ADT-06**: Unary constructor compilation — {tag, payload_ptr} heap block
- [x] **ADT-07**: Multi-arg constructor — tuple payload wrapping
- [x] **ADT-08**: ConstructorPat (nullary) — tag comparison only
- [x] **ADT-09**: ConstructorPat (unary) — tag + GEP + load + sub-pattern
- [x] **ADT-10**: GADT constructor — same as ADT after type check
- [x] **ADT-11**: First-class constructor — unary ctor as lambda closure
- [x] **ADT-12**: Nested ADT pattern matching — multi-level GEP chains

### Records
- [x] **REC-01**: RecordDecl processing — field name -> index mapping
- [x] **REC-02**: RecordExpr compilation — GC_malloc(n*8) + field store
- [x] **REC-03**: FieldAccess compilation — GEP(fieldIndex) + load
- [x] **REC-04**: RecordUpdate compilation — new block + copy + override
- [x] **REC-05**: SetField compilation — GEP + store, returns unit
- [x] **REC-06**: RecordPat compilation — field-name structural match

### Exception Handling
- [x] **EXN-01**: C runtime — LangExnFrame, lang_try_push, _setjmp, lang_try_exit, lang_throw
- [x] **EXN-02**: Raise compilation — @lang_throw + llvm.unreachable
- [x] **EXN-03**: TryWith compilation — setjmp branch + handler dispatch + merge
- [x] **EXN-04**: Nested try-with — correct handler stack push/pop
- [x] **EXN-05**: Exception with payload — string payload through throw/catch
- [x] **EXN-06**: Unhandled exception — runtime abort with message
- [x] **EXN-07**: Exception re-raise — handler miss -> lang_throw
- [x] **EXN-08**: Exception in handler — raise inside handler arm propagates
