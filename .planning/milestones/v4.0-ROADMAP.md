## Milestone Summary

**Key Decisions:**
- ADT layout: 16-byte GC_malloc block, slot 0 = i64 tag, slot 1 = payload stored directly
- Record flat layout: GC_malloc(n*8), no tag prefix (unlike ADT)
- setjmp/longjmp for exception unwinding, inline _setjmp for ARM64 PAC compatibility
- Uniform closure ABI (ptr, i64) -> i64 with ptrtoint/inttoptr for type bridging
- MatchCompiler decision tree shared between Match and TryWith (duplicated code)

**Issues Resolved:**
- ARM64 PAC: out-of-line setjmp wrapper freed stack before longjmp; fixed by inline _setjmp
- Nested try-with in Let bindings: block terminator detection + patching
- Dead merge blocks when all handler arms are noreturn: predecessor existence check

**Technical Debt Incurred:**
- ExnTags map populated but never read (all resolution goes through TypeEnv)
- elaborateModule duplicates externalFuncs list (dead code in production)
- emitDecisionTree duplicated in TryWith case (cannot share closures)
