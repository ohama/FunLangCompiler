  <action>
1. Add a `TypeInfo` record type near the top of Elaboration.fs (after FuncSignature):
   ```fsharp
   type TypeInfo = { Tag: int; Arity: int }
   ```

2. Add three fields to `ElabEnv`:
   ```fsharp
   TypeEnv:    Map<string, TypeInfo>         // constructor name -> tag + arity
   RecordEnv:  Map<string, Map<string, int>> // record type name -> (field name -> index)
   ExnTags:    Map<string, int>              // exception ctor name -> tag index
   ```

3. Update `emptyEnv` to include `TypeEnv = Map.empty; RecordEnv = Map.empty; ExnTags = Map.empty`.

4. Implement `prePassDecls`:
   ```fsharp
   let private prePassDecls (decls: Ast.Decl list) =
   ```
   - For each `Ast.TypeDecl(Ast.TypeDecl(_, _, ctors, _))`: iterate constructors with `List.iteri`, adding each to `typeEnv` as `{ Tag = idx; Arity = (0 if None, 1 if Some) }`. Handle both `ConstructorDecl` and `GadtConstructorDecl`.
   - For each `Ast.RecordTypeDecl(Ast.RecordDecl(typeName, _, fields, _))`: build a `Map<string, int>` from field name to index via `List.mapi`, add to `recordEnv` keyed by `typeName`.
   - For each `Ast.ExceptionDecl(name, dataType, _)`: assign a unique tag using a `ref int` counter (NOT Map.count), add to `exnTags`.
   - Return `(typeEnv, recordEnv, exnTags)`.

   IMPORTANT: Use double-level pattern matching for TypeDecl -- outer `Ast.Decl.TypeDecl` wraps inner `Ast.TypeDecl(...)`. Same for RecordTypeDecl wrapping RecordDecl. See Research Pitfall 5.

5. Implement `elaborateProgram`:
   ```fsharp
   let elaborateProgram (ast: Ast.Module) : MlirModule =
   ```
   - Extract `Decl list` from the module variant (Module, NamedModule, NamespacedModule -> decls; EmptyModule -> []).
   - Call `prePassDecls` to get `(typeEnv, recordEnv, exnTags)`.
   - Build env: `{ emptyEnv() with TypeEnv = typeEnv; RecordEnv = recordEnv; ExnTags = exnTags }`.
   - Extract the main expression from decls. CRITICAL: First, empirically determine what `Parser.parseModule` produces for bare expression inputs like `42` or `let x = 5 in x + 1` by examining the parser. The most likely scenario is that bare expressions become `LetDecl` or similar wrappers. Design `extractMainExpr` accordingly -- it should fold/chain LetDecls into nested Let expressions, with the last decl's body as the final expression.
   - Call the same elaboration logic as `elaborateModule` (reuse the body of elaborateModule, either by calling it or extracting shared logic into a helper like `elaborateModuleFromEnv`).

   Keep `elaborateModule` intact (do not delete it) so the code compiles at every step, but the CLI will no longer call it.
