```fsharp
    // Phase 19: Raise — construct exception value, call @lang_throw, terminate block
    | Raise(exnExpr, _) ->
        let (exnVal, exnOps) = elaborateExpr env exnExpr
        // exnVal is a Ptr to an ADT DataValue block (e.g., {tag=0, payload=LangString*} for Failure "msg")
        // Call @lang_throw(exnVal) — noreturn void call
        // Emit llvm.unreachable after the call to terminate the block
        let deadVal = { Name = freshName env; Type = I64 }
        (deadVal, exnOps @ [ LlvmCallVoidOp("@lang_throw", [exnVal]); LlvmUnreachableOp ])
```

Key points:
- `exnExpr` is already a `Constructor` expression (e.g., `Constructor("Failure", Some(String "boom"))`), so `elaborateExpr env exnExpr` produces the ADT DataValue block pointer via the existing Constructor case. The prePassDecls fix from 19-01 ensures `Failure` is in TypeEnv.
- `deadVal` is a placeholder — `elaborateExpr` must return `(MlirValue * MlirOp list)`, but after `llvm.unreachable` no value is ever used. Return a fresh I64 name that will never be referenced.
- `LlvmCallVoidOp("@lang_throw", [exnVal])` uses the existing void-call pattern (same as `@lang_match_failure` but with an argument).
- `LlvmUnreachableOp` terminates the basic block (same as after `@lang_match_failure` in the Match fail block).

**IMPORTANT:** The `Raise` case must appear BEFORE any catch-all wildcard pattern. Check for `| _ ->` at the end of `elaborateExpr` and place the new case before it.
