Add a new `TryWith` case to `elaborateExpr`, placing it immediately after the `Raise` case added in 19-02.

The TryWith elaboration follows a multi-block CFG pattern similar to `Match`, but with setjmp branching at the entry. Here is the complete control flow:

```
[entry ops — inline in current block]
  %frame_size = arith.constant 272 : i64
  %frame = llvm.call @GC_malloc(%frame_size)
  %sjr = llvm.call @lang_try_enter(%frame)
  %zero = arith.constant 0 : i64
  %is_normal = arith.cmpi eq, %sjr, %zero
  cf.cond_br %is_normal, ^try_body, ^exn_caught

^try_body:
  [elaborate body expr]
  llvm.call @lang_try_exit()       // pop handler on normal path
  cf.br ^merge(%body_result)

^exn_caught:
  llvm.call @lang_try_exit()       // C-16: pop handler BEFORE dispatch
  %exn = llvm.call @lang_current_exception()
  [MatchCompiler decision tree over handlers, scrutinee = %exn]
  [each handler Leaf: bind vars, elaborate arm body, cf.br ^merge(%arm_result)]
  [Fail case: call @lang_throw(%exn) + llvm.unreachable]  // re-raise

^merge(%result):
  [value used by outer expression]
```

Implementation in F#:

```fsharp
    // Phase 19: TryWith — setjmp/longjmp exception handling
    | TryWith(bodyExpr, clauses, _) ->
        let mergeLabel     = freshLabel env "try_merge"
        let tryBodyLabel   = freshLabel env "try_body"
        let exnCaughtLabel = freshLabel env "exn_caught"
        let exnFailLabel   = freshLabel env "exn_fail"

        // --- Entry ops: allocate frame, call lang_try_enter, branch ---
        let frameSizeVal = { Name = freshName env; Type = I64 }
        let framePtr     = { Name = freshName env; Type = Ptr }
        let sjResult     = { Name = freshName env; Type = I64 }
        let zero64       = { Name = freshName env; Type = I64 }
        let isNormal     = { Name = freshName env; Type = I1 }
        let entryOps = [
            ArithConstantOp(frameSizeVal, 272L)
            LlvmCallOp(framePtr, "@GC_malloc", [frameSizeVal])
            LlvmCallOp(sjResult, "@lang_try_enter", [framePtr])
            ArithConstantOp(zero64, 0L)
            ArithCmpIOp(isNormal, "eq", sjResult, zero64)
            CfCondBrOp(isNormal, tryBodyLabel, [], exnCaughtLabel, [])
        ]

        // --- ^try_body block: elaborate body, pop handler, branch to merge ---
        let (bodyVal, bodyOps) = elaborateExpr env bodyExpr
        let resultType = ref bodyVal.Type
        let tryBodyBlock = {
            Label = Some tryBodyLabel
            Args  = []
            Body  = bodyOps @ [
                LlvmCallVoidOp("@lang_try_exit", [])
                CfBrOp(mergeLabel, [bodyVal])
            ]
        }
        env.Blocks.Value <- env.Blocks.Value @ [tryBodyBlock]

        // --- ^exn_caught block: pop handler, get exception, dispatch handlers ---
        // Build the MatchCompiler decision tree from handler clauses
        // (same pattern as Match case, lines ~903-1216)

        // Expand OrPat in handlers (same as Match)
        let clauses =
            clauses |> List.collect (fun (pat, guard, body) ->
                match pat with
                | OrPat(alts, _) -> alts |> List.map (fun altPat -> (altPat, guard, body))
                | _ -> [(pat, guard, body)]

        let exnPtrVal = { Name = freshName env; Type = Ptr }
        let exnPreambleOps = [
            LlvmCallVoidOp("@lang_try_exit", [])   // C-16: pop before dispatch
            LlvmCallOp(exnPtrVal, "@lang_current_exception", [])
        ]

        // Build arms for MatchCompiler
        let arms = clauses |> List.mapi (fun i (pat, guardOpt, _body) -> (pat, guardOpt.IsSome, i))
        let rootAcc = MatchCompiler.Root exnPtrVal.Name
        let tree = MatchCompiler.compile rootAcc arms

        // Accessor cache and resolver — COPY the exact same resolveAccessor/resolveAccessorTyped/

        // - scrutineeTypeForTag

        // - ensureConsFieldTypes / ensureAdtFieldTypes / ensureRecordFieldTypes
        // - emitDecisionTree (recursive function)

        // KEY DIFFERENCES from Match:

        // 2. The Fail case calls @lang_throw(exnPtrVal) + LlvmUnreachableOp (re-raise)
        //    instead of @lang_match_failure + LlvmUnreachableOp
        // 3. Body blocks branch to mergeLabel with [armVal] (same as Match)
        // 4. resultType is shared with the try_body block

        // env.Blocks.Value <- env.Blocks.Value @ [exnCaughtBlock]

        //   CfBrOp(exnFailLabel, [])
        // with the fail block:
        // env.Blocks.Value <- env.Blocks.Value @ [{
        //     Label = Some exnFailLabel; Args = [];
        //     Body = [ LlvmCallVoidOp("@lang_throw", [exnPtrVal]); LlvmUnreachableOp ]
```

**CONCRETE IMPLEMENTATION GUIDANCE:**

Since the local helper functions from the Match case must be duplicated, here is the recommended approach:

1. Copy the entire block of local functions from the Match case (resolveAccessor, resolveAccessorTyped, scrutineeTypeForTag, emitCtorTest, ensureConsFieldTypes, ensureAdtFieldTypes, ensureRecordFieldTypes, emitDecisionTree) into the TryWith case.

2. Make these modifications to the duplicated code:
   - `accessorCache.[rootAcc] <- exnPtrVal` (instead of scrutVal)
   - In `emitDecisionTree`, the `Fail` case emits:
     ```fsharp
     | MatchCompiler.Fail ->
         [ CfBrOp(exnFailLabel, []) ]
     ```
   - The `exnFailLabel` block contains:
     ```fsharp
     { Label = Some exnFailLabel; Args = [];
       Body = [ LlvmCallVoidOp("@lang_throw", [exnPtrVal]); LlvmUnreachableOp ] }
     ```
   - In `emitDecisionTree`, the `Leaf` case uses `clauses` (the TryWith handler clauses, not the match clauses). The body block branches to `mergeLabel`.
   - `resultType` is shared — it's set by the first Leaf OR the try body (whichever is reached first during emission). Use the same `ref` initialized from `bodyVal.Type`.

3. After emitting the decision tree:
   - Add the exn_caught block to `env.Blocks.Value`
   - Add the exn_fail block to `env.Blocks.Value`
   - Add the merge block to `env.Blocks.Value`

4. Return `(mergeArg, entryOps)` — the entry ops contain the frame allocation, setjmp call, and conditional branch. The side blocks (try_body, exn_caught, exn_fail, merge) are appended to `env.Blocks.Value`.

**CRITICAL: Order of block emission matters.** The merge block MUST be the LAST block added, because the `elaborateProgram` function appends `ReturnOp [resultVal]` to the last block in `env.Blocks.Value`. If TryWith is the outermost expression, merge must be last so the return op goes there.

**Troubleshooting common failures:**

1. **MLIR type mismatch in merge block:** The try body and all handler arms must produce the same MlirType. If the try body produces I64 but a handler arm produces Ptr (or vice versa), the merge block argument type will mismatch. Fix: ensure `resultType` is set correctly.

2. **"Fatal: non-exhaustive match" instead of "Fatal: unhandled exception":** This means the Fail case in emitDecisionTree is branching to `@lang_match_failure` instead of `@lang_throw`. Verify the TryWith version of emitDecisionTree uses `@lang_throw(exnPtrVal)` for Fail.

3. **Segfault on nested try-with:** Likely the handler stack pop ordering is wrong. Verify `lang_try_exit` is called in BOTH `^try_body` (normal path) and `^exn_caught` (exception path), and ONLY once per path.

4. **KeyNotFoundException on exception constructor name:** The prePassDecls fix from 19-01 should have added exception constructors to TypeEnv. If this test fails, check that fix.

5. **Nested try test returns wrong value:** May be a block ordering issue. The merge block must be the last block emitted for TryWith if it's the outermost expression.
