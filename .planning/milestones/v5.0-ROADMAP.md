## Milestone Summary

**Key Decisions:**
- GC ref cell (8-byte GC_malloc'd box) for mutable variables
- Closure captures mutable var as Ptr (shared ref cell, not value copy)
- One-block array layout: GC_malloc((n+1)*8), slot 0 = length
- LlvmGEPDynamicOp for runtime-computed array index
- Hashtable: chained buckets in C runtime, murmurhash3 fmix64, i64 key identity
- lang_throw for OOB/missing-key exceptions (catchable by try/with)
- Array HOFs via C runtime closure callbacks (LangClosureFn typedef)
- array_fold uses two-call curried pattern per iteration

**Technical Debt:**
- Hashtable string keys use i64 identity equality (not content equality)
- hashtable_keys order unspecified (bucket-iteration order)
