<task type="auto">
  <name>Task 2: Implement LetMut, Var deref, Assign, and LetMutDecl elaboration</name>
  <files>src/FunLangCompiler.Compiler/Elaboration.fs</files>
  <action>
  Four changes in the `elaborateExpr` function and `extractMainExpr`:

  1. **LetMut case** — add to `elaborateExpr` match (near other Let cases):
  ```fsharp
  | LetMut (name, initExpr, bodyExpr, _) ->
      let (initVal, initOps) = elaborateExpr env initExpr
      let sizeVal  = { Name = freshName env; Type = I64 }
      let cellPtr  = { Name = freshName env; Type = Ptr }
      let allocOps = [
          ArithConstantOp(sizeVal, 8L)
          LlvmCallOp(cellPtr, "@GC_malloc", [sizeVal])
          LlvmStoreOp(initVal, cellPtr)
      ]
      let env' = { env with
                      Vars        = Map.add name cellPtr env.Vars
                      MutableVars = Set.add name env.MutableVars }
      let (bodyVal, bodyOps) = elaborateExpr env' bodyExpr
      (bodyVal, initOps @ allocOps @ bodyOps)
  ```

  2. **Var case** — modify existing Var case (line ~332) to check MutableVars:
  ```fsharp
  | Var (name, _) ->
      match Map.tryFind name env.Vars with
      | Some v ->
          if Set.contains name env.MutableVars then
              let loaded = { Name = freshName env; Type = I64 }
              (loaded, [LlvmLoadOp(loaded, v)])
          else
              (v, [])
      | None -> failwithf "Elaboration: unbound variable '%s'" name
  ```

  3. **Assign case** — add to `elaborateExpr` match:
  ```fsharp
  | Assign (name, valExpr, _) ->
      let (newVal, valOps) = elaborateExpr env valExpr
      let cellPtr =
          match Map.tryFind name env.Vars with
          | Some v -> v
          | None -> failwithf "Elaboration: unbound mutable variable '%s' in Assign" name
      let unitVal = { Name = freshName env; Type = I64 }
      (unitVal, valOps @ [LlvmStoreOp(newVal, cellPtr); ArithConstantOp(unitVal, 0L)])
  ```
  CRITICAL: Do NOT update env.Vars in Assign. The cell pointer is fixed; only the cell content changes via LlvmStoreOp.

  4. **LetMutDecl in extractMainExpr** (line ~2049):
  - Update the `exprDecls` filter to include `Ast.Decl.LetMutDecl _`:
    ```fsharp
    | Ast.Decl.LetDecl _ | Ast.Decl.LetRecDecl _ | Ast.Decl.LetMutDecl _ -> true
    ```
  - Add cases in the `build` function:
    ```fsharp
    | [Ast.Decl.LetMutDecl(name, body, _)] -> LetMut(name, body, Var(name, s), s)
    ```
    And for multi-decl:
    ```fsharp
    | Ast.Decl.LetMutDecl(name, body, _) :: rest ->
        LetMut(name, body, build rest, s)
    ```
  </action>
  <verify>Run `dotnet build src/FunLangCompiler.Compiler/FunLangCompiler.Compiler.fsproj` — must compile. Then run a quick smoke test:
  Create a temp file with `let mut x = 5 in x` and compile+run it — should exit with code 5.</verify>
  <done>LetMut allocates GC ref cell. Var transparently dereferences mutable names. Assign stores to ref cell and returns unit. LetMutDecl desugars to LetMut in extractMainExpr.</done>
