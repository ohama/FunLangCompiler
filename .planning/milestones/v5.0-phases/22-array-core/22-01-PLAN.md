```c
/* Phase 22: Forward declaration for array-list conversion */
typedef struct LangCons LangCons;

/* Phase 22: Array builtins */
int64_t* lang_array_create(int64_t n, int64_t default_val);
void lang_array_bounds_check(int64_t* arr, int64_t i);

LangCons* lang_array_to_list(int64_t* arr);
```

IMPORTANT: Since `LangCons` is already defined with `typedef struct LangCons { ... } LangCons;` in `lang_runtime.c`, a forward `typedef struct LangCons LangCons;` in the header is compatible. However, the header MUST use `typedef struct LangCons LangCons;` (not just `struct LangCons;`) to match the typedef in the .c file.

2. In `lang_runtime.c`, add all four functions at the end of the file (before any closing comments):

**lang_array_create(n, default_val):**
- If `n < 0`, call `lang_failwith("Array.create: negative size")` and return NULL.
- Allocate `(n + 1) * sizeof(int64_t)` bytes via `GC_malloc` (NOT `GC_malloc_atomic` -- array may hold GC pointers).
- Set `arr[0] = n` (length in slot 0).
- Fill slots `arr[1]` through `arr[n]` with `default_val`.
- Return `arr`.

**lang_array_bounds_check(arr, i):**
- Read `len = arr[0]`.
- If `i < 0 || i >= len`:
  1. Format the error message `"Array index %ld out of bounds (length %ld)"` via snprintf into a stack buffer (e.g., `char buf[128]`).
  2. Allocate a `LangString*` for the error message: `LangString* msg = (LangString*)GC_malloc(sizeof(LangString)); msg->length = strlen(buf); msg->data = ...(GC_malloc copy of buf)`.
  3. Call `lang_throw((void*)msg)` -- NOT `lang_failwith`. This is CRITICAL: `lang_failwith` calls `exit(1)` and makes the error uncatchable. `lang_throw` uses `longjmp` to the nearest `try/with` handler, making OOB errors catchable exceptions.
- Otherwise return (no-op).

The `LangString` struct is already defined at the top of `lang_runtime.c` as `typedef struct { int64_t length; char* data; } LangString;`. Use it directly.

**lang_array_of_list(list):**
- First pass: count length by traversing cons list (while `cur != NULL`).
- Allocate `(n + 1) * sizeof(int64_t)` via `GC_malloc`.
- Set `arr[0] = n`.
- Second pass: copy elements from cons cells into `arr[1]` through `arr[n]`.
- Return `arr`.
- Handle empty list (NULL input): allocate length-0 array (`arr[0] = 0`, no element slots beyond that).

**lang_array_to_list(arr):**
- Read `n = arr[0]`.
- Build cons list by iterating BACKWARDS from `arr[n]` down to `arr[1]` (reverse iteration produces forward-order list).
- Each iteration: `GC_malloc(sizeof(LangCons))`, set `cell->head = arr[i]`, `cell->tail = head`, `head = cell`.
- Return `head` (NULL for empty array).
