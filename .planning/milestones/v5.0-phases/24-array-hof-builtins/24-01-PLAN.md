1. **lang_array_iter** (ARR-08): Loop i from 1 to n, call `fn(closure, arr[i])`, discard return. Returns void.
```c
void lang_array_iter(void* closure, int64_t* arr) {
    int64_t n = arr[0];
    LangClosureFn fn = *(LangClosureFn*)closure;
    for (int64_t i = 1; i <= n; i++) {
        fn(closure, arr[i]);
    }
}
```

2. **lang_array_map** (ARR-09): Allocate output array via GC_malloc((n+1)*8), set out[0]=n, loop calling `out[i] = fn(closure, arr[i])`.
```c
int64_t* lang_array_map(void* closure, int64_t* arr) {
    int64_t n = arr[0];
    int64_t* out = (int64_t*)GC_malloc((size_t)((n + 1) * 8));
    out[0] = n;
    LangClosureFn fn = *(LangClosureFn*)closure;
    for (int64_t i = 1; i <= n; i++) {
        out[i] = fn(closure, arr[i]);
    }
    return out;
}
```

3. **lang_array_fold** (ARR-10): CRITICAL — the fold function is a curried two-arg closure. Each iteration requires TWO closure calls:
   - First call: `int64_t partial = fn(closure, acc)` returns an intermediate closure (as ptrtoint i64)
   - Second call: Cast partial to `void*`, load fn_ptr, call `fn2(partial_ptr, arr[i])` to get new acc
```c
int64_t lang_array_fold(void* closure, int64_t init, int64_t* arr) {
    int64_t n = arr[0];
    LangClosureFn fn = *(LangClosureFn*)closure;
    int64_t acc = init;
    for (int64_t i = 1; i <= n; i++) {
        int64_t partial = fn(closure, acc);
        void* partial_ptr = (void*)partial;
        LangClosureFn fn2 = *(LangClosureFn*)partial_ptr;
        acc = fn2(partial_ptr, arr[i]);
    }
    return acc;
}
```

4. **lang_array_init** (ARR-11): Negative size check via lang_failwith. Allocate output array. Loop i from 0 to n-1 (zero-based logical index), store at out[i+1] (physical slot).
```c
int64_t* lang_array_init(int64_t n, void* closure) {
    if (n < 0) {
        lang_failwith("array_init: negative size");
        return NULL;
    }
    int64_t* out = (int64_t*)GC_malloc((size_t)((n + 1) * 8));
    out[0] = n;
    LangClosureFn fn = *(LangClosureFn*)closure;
    for (int64_t i = 0; i < n; i++) {
        out[i + 1] = fn(closure, i);
    }
    return out;
}
```

Key points to NOT get wrong:
- fold uses TWO closure calls per iteration (curried binary function)
- init uses zero-based i for the closure arg, stores at i+1
- Use GC_malloc (not GC_malloc_atomic) — output arrays may contain pointers
- iter returns void, not int64_t
