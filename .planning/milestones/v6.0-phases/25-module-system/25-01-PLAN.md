  <name>Task 1: prePassDecls recursion + flattenDecls + extractMainExpr changes</name>
  <files>src/FunLangCompiler.Compiler/Elaboration.fs</files>
  <action>
  Three changes in Elaboration.fs:

  **1. Make prePassDecls recursive, accept shared exnCounter, and add ModuleDecl/NamespaceDecl arms (line ~2317):**
  - Change `let private prePassDecls` to `let rec private prePassDecls (exnCounter: int ref) (decls: Ast.Decl list)`
  - The exnCounter parameter is passed through ALL recursive calls so that exception tags are globally unique. Without this, each recursive call would create its own counter starting at 0, causing tag collisions between module-level and top-level exceptions.
  - Update the existing body to use the passed-in exnCounter instead of creating a local `let exnCounter = ref 0`.
  - Add two new arms BEFORE the `| _ -> ()` wildcard:
    ```
    | Ast.Decl.ModuleDecl(_, innerDecls, _) ->
        let (iT, iR, iE) = prePassDecls exnCounter innerDecls
        typeEnv  <- Map.fold (fun acc k v -> Map.add k v acc) typeEnv  iT
        recordEnv <- Map.fold (fun acc k v -> Map.add k v acc) recordEnv iR
        exnTags  <- Map.fold (fun acc k v -> Map.add k v acc) exnTags  iE
    | Ast.Decl.NamespaceDecl(_, innerDecls, _) ->
        let (iT, iR, iE) = prePassDecls exnCounter innerDecls
        typeEnv  <- Map.fold (fun acc k v -> Map.add k v acc) typeEnv  iT
        recordEnv <- Map.fold (fun acc k v -> Map.add k v acc) recordEnv iR
        exnTags  <- Map.fold (fun acc k v -> Map.add k v acc) exnTags  iE
    ```
  - Update the single call site in `elaborateProgram` (line ~2399) to pass `ref 0` as the first argument: `prePassDecls (ref 0) decls`

  **2. Add flattenDecls helper (insert BEFORE extractMainExpr, around line ~2350):**
  ```
  let rec private flattenDecls (decls: Ast.Decl list) : Ast.Decl list =
      decls |> List.collect (fun d ->
          match d with
          | Ast.Decl.ModuleDecl(_, innerDecls, _) -> flattenDecls innerDecls
          | Ast.Decl.NamespaceDecl(_, innerDecls, _) -> flattenDecls innerDecls
          | _ -> [d])
  ```

  **3. Modify extractMainExpr (line ~2355):**
  - Add `let flatDecls = flattenDecls decls` as the first line after `let s = unknownSpan`
  - Change `decls |> List.filter` to `flatDecls |> List.filter`
  - Add `| Ast.Decl.LetPatDecl _ -> true` to the filter (alongside LetDecl, LetRecDecl, LetMutDecl)
  - Add LetPatDecl arms to the `build` function:
    - Single: `| [Ast.Decl.LetPatDecl(pat, body, _)] -> LetPat(pat, body, Number(0, s), s)`
    - With rest: `| Ast.Decl.LetPatDecl(pat, body, _) :: rest -> LetPat(pat, body, build rest, s)`
    Place these BEFORE the `| _ :: rest -> build rest` wildcard arm.

  OpenDecl requires NO additional handling in extractMainExpr — it hits the wildcard `| _ -> false` in the filter and is correctly skipped. NamespaceDecl is handled by flattenDecls (which recurses into it to extract inner decls), so by the time the filter runs, NamespaceDecl nodes have been replaced by their inner declarations. Neither OpenDecl nor NamespaceDecl needs special arms in the filter or build steps.

          raise (ModError 42)

  Expected output: `42\n99\n0`
  This test exercises the shared exnCounter — ModError gets tag 0 (inside module), TopError gets tag 1 (at top level). If exnCounter were not shared, both would get tag 0 and catch would mismatch.
