  <name>Task 1: Qualified name desugar guard in elaborateExpr</name>
  <files>src/FunLangCompiler.Compiler/Elaboration.fs</files>
  <action>
  Add a new match arm in elaborateExpr BEFORE the existing `| FieldAccess(recExpr, fieldName, _) ->` arm (currently at line ~1777). The ordering is critical — F# matches top-to-bottom, so the new guard must come first.

  Insert this arm:
  ```fsharp
  // Phase 25: MOD-05 — desugar qualified module access M.member
  | FieldAccess(Constructor(_, None, _), memberName, span) ->
      if Map.containsKey memberName env.TypeEnv then
          // memberName is a known constructor (ADT or exception)
          elaborateExpr env (Constructor(memberName, None, span))
      else
          // memberName is a regular value binding
          elaborateExpr env (Var(memberName, span))
  ```

  This handles: M.x (value), M.f (function), M.Ctor (constructor), M.ExnName (exception constructor).

  The guard works because:
  1. Qualified names parse as FieldAccess(Constructor(modName, None, span), memberName, span)
  2. After flattenDecls (Plan 25-01), all module bindings are in scope as plain names
  3. The TypeEnv check distinguishes constructors from value bindings
  4. Delegating to elaborateExpr with Var or Constructor reuses all existing resolution logic

  Also consider adding a two-level qualified access guard for M.Sub.x if the research recommends it. The pattern is:
  ```fsharp
  | FieldAccess(FieldAccess(Constructor(_, None, _), _, _), memberName, span) ->
      if Map.containsKey memberName env.TypeEnv then
          elaborateExpr env (Constructor(memberName, None, span))
      else
          elaborateExpr env (Var(memberName, span))
  ```
  Only add this if a test requires it. For v6.0, single-level qualified access (M.x) is sufficient.
  </action>
