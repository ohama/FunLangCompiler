  <action>
Add 6 C functions to lang_runtime.c, appending after the existing lang_array_init function:

1. `LangString* lang_file_read(LangString* path)` -- Opens file with "rb", seeks to get size, GC_malloc's content buffer, fread's contents, null-terminates, returns new LangString*. On fopen failure, constructs error message "read_file: file not found: {path}" as LangString* via GC_malloc, calls lang_throw((void*)msg). NOTE: In the fopen failure branch, f == NULL so there is nothing to close before lang_throw. The pattern is correct as written.

2. `void lang_file_write(LangString* path, LangString* content)` -- fopen with "wb", fwrite content->data for content->length bytes, fclose. Silently return on fopen failure (no throw).

3. `void lang_file_append(LangString* path, LangString* content)` -- Same as write but fopen with "ab".

4. `int64_t lang_file_exists(LangString* path)` -- fopen with "r", if non-NULL fclose and return 1, else return 0.

5. `void lang_eprint(LangString* s)` -- fwrite(s->data, 1, s->length, stderr) then fflush(stderr).

6. `void lang_eprintln(LangString* s)` -- fwrite(s->data, 1, s->length, stderr) then fputc('\n', stderr) then fflush(stderr).

Add corresponding declarations in lang_runtime.h before the `#endif`, after the existing lang_array_init declaration. Include the `<string.h>` header if not already present in lang_runtime.c (needed for strlen/memcpy in the error message construction for lang_file_read). Check that `<stdio.h>` is already included (it should be for fprintf/fopen etc.).

Follow the exact code from 26-RESEARCH.md Pattern 1 (lang_file_read), Pattern 2 (write/append/eprint/eprintln), and Pattern 3 (file_exists).
