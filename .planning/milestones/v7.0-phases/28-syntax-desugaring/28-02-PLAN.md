<objective>
Implement IndexGet and IndexSet elaboration so that `arr.[i]`, `arr.[i] <- v`, `ht.[key]`, and `ht.[key] <- v` compile to native code.

Purpose: The FunLang parser produces IndexGet/IndexSet AST nodes for `.[...]` syntax. The type checker verifies types but does NOT rewrite these nodes. The backend must handle them. Since the backend has no type info, we use runtime dispatch: a tag field in LangHashtable distinguishes hashtables from arrays, and new C runtime helpers `lang_index_get`/`lang_index_set` dispatch to the correct operation.

Output: Working IndexGet/IndexSet compilation + 5 E2E tests covering IDX-01 through IDX-04

**Step 3 — Add dispatch functions (lang_runtime.c):**

```c
int64_t lang_index_get(void* collection, int64_t index) {
    int64_t first_word = ((int64_t*)collection)[0];
    if (first_word < 0) {
        // Hashtable: tag is -1
        return lang_hashtable_get((LangHashtable*)collection, index);
    } else {
        // Array: first word is non-negative length
        int64_t* arr = (int64_t*)collection;
        lang_array_bounds_check(arr, index);
        return arr[index + 1];
    }
}

void lang_index_set(void* collection, int64_t index, int64_t value) {
    int64_t first_word = ((int64_t*)collection)[0];
    if (first_word < 0) {
        // Hashtable: tag is -1
        lang_hashtable_set((LangHashtable*)collection, index, value);
    } else {
        // Array: first word is non-negative length
        int64_t* arr = (int64_t*)collection;
        lang_array_bounds_check(arr, index);
        arr[index + 1] = value;
    }
}
```

The dispatch logic: arrays store length (>= 0) at offset 0; hashtables store tag = -1 at offset 0. This is safe because array lengths are always non-negative.

IMPORTANT: Do NOT change any existing function signatures. The tag field addition to LangHashtable only affects the struct layout; all existing C functions that take `LangHashtable*` will still work correctly because they access fields by name, not raw offset.

    | IndexSet (collExpr, idxExpr, valExpr, _) ->
        let (collVal, collOps) = elaborateExpr env collExpr
        let (idxVal, idxOps) = elaborateExpr env idxExpr
        let (valVal, valOps) = elaborateExpr env valExpr
        let idxI64 =

        let valI64 =
            if valVal.Type = I64 then (valVal, [])
            else let v = { Name = freshName env; Type = I64 } in (v, [ArithExtuIOp(v, valVal)])
        let (idxV, idxCoerce) = idxI64
        let (valV, valCoerce) = valI64

NOTE on coercion: The index and value may come in as I1 (bool) or Ptr types. For index, we coerce I1 to I64 (same as array_get pattern). For value, same coercion.
