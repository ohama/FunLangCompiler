  **1. freeVars case (next to the WhileExpr case added in plan 01, before `| _ -> Set.empty`):**

  ```fsharp
  | ForExpr (var, startExpr, _, stopExpr, body, _) ->
      Set.unionMany [
          freeVars boundVars startExpr
          freeVars boundVars stopExpr
          freeVars (Set.add var boundVars) body   // var is bound inside body
      ]
  ```

  This covers LOOP-05 for ForExpr.

  **2. elaborateExpr case for ForExpr (add after the WhileExpr case):**

  Use the block-argument pattern for the loop counter. The CFG shape is:

  ```
  entry fragment:
    <elaborate start into %start : i64>
    <elaborate stop into %stop : i64>
    arith.constant %unitConst = 0 : i64       -- unit value, defined here for domination
    cf.br ^for_header(%start : i64)

  ^for_header(%i : i64):
    arith.cmpi "sle"/%sge", %i, %stop : i64   -- sle for `to`, sge for `downto`
    cf.cond_br %cmp, ^for_body, ^for_exit(%unitConst)

  ^for_body:
    <elaborate body with env.Vars[var] = %i, NOT in MutableVars>
    arith.constant %one = 1 : i64
    arith.addi/%subi %next, %i, %one           -- addi for `to`, subi for `downto`
    cf.br ^for_header(%next : i64)

  ^for_exit(%exitArg : i64):
    (empty body -- patched by LetPat continuation)
  ```

  Implementation details:
  - `freshLabel env "for_header"`, `freshLabel env "for_body"`, `freshLabel env "for_exit"`.
  - Elaborate `startExpr` -> `(startVal, startOps)` and `stopExpr` -> `(stopVal, stopOps)`.
  - Create `iArg = { Name = freshName env; Type = I64 }` as the block argument of `^for_header`.
  - Create `bodyEnv = { env with Vars = Map.add var iArg env.Vars }` -- do NOT add `var` to `MutableVars`. This ensures the for-variable is immutable (LOOP-04).
  - Elaborate `bodyExpr` with `bodyEnv` -> `(_bodyVal, bodyOps)`, discard the value.
  - Comparison predicate: `if isTo then "sle" else "sge"`.
  - Increment op: `if isTo then ArithAddIOp(nextVal, iArg, oneConst) else ArithSubIOp(nextVal, iArg, oneConst)`.
  - `unitConst` via `ArithConstantOp(unitConst, 0L)` placed in the ENTRY fragment ops (not in header or body). This ensures unitConst dominates all blocks.
  - `CfCondBrOp(cmpVal, bodyLabel, [], exitLabel, [unitConst])` in header -- passes unitConst to exit on false.
  - `CfBrOp(headerLabel, [nextVal])` in body -- passes incremented counter back to header.
  - `CfBrOp(headerLabel, [startVal])` in entry fragment -- enters loop with start value.
  - Push 3 blocks to `env.Blocks.Value`: header (with `Args = [iArg]`), body, exit (with `Args = [exitArg]`, `Body = []`).
  - Return `(exitArg, startOps @ stopOps @ [ArithConstantOp(unitConst, 0L); CfBrOp(headerLabel, [startVal])])`.

  IMPORTANT notes:
  - `^for_header` has `Args = [iArg]` -- this is the block argument that carries the loop counter.
  - The `stopVal` is an SSA value defined in the entry fragment. It dominates `^for_header` (entry branches to header), so using `stopVal` in the header's `ArithCmpIOp` is SSA-valid.
  - `oneConst` and `nextVal` are defined in the body block.
  - Exit block `Body = []` for LetPat patching compatibility.
  - Do NOT add var to `env.MutableVars` -- the for-variable must be immutable.
