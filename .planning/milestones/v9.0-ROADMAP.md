## Milestone Summary

**Key Decisions:**
- hashtable_count uses inline GEP at field index 2 (no C function needed)
- E2E tests use `println (to_string ...)` — `printfn "%d"` does not exist in elaborator
- Struct typedefs defined only in lang_runtime.h (prevents clang redefinition errors)
- StringBuilder/MutableList growth uses GC_malloc+memcpy (never realloc)
- HashSet reuses lang_ht_hash murmurhash3
- CollectionVars: Map<string, CollectionKind> for compile-time for-in dispatch
- Module-qualified naming (Option_map, String_endsWith) eliminates MLIR symbol collisions
- CLI prelude auto-loading walks up from input file directory

**Issues Resolved:**
- 11 compiler bugs fixed during Phase 35 (closure captures, accessor cache, LetRec, type coercions)
- ForInExpr TuplePat I64→Ptr coercion generalized in LetPat arm
- Module name collision resolved via qualified naming in flattenDecls

**Issues Deferred:**
- Two-sequential-if MLIR limitation (empty entry block)
- for-in mutable capture segfault (pre-existing)
- Hashtable string keys (C hashtable uses int64_t key ABI)
- List take/drop/zip absent (blocked by if-else-match MLIR bug)

**Technical Debt:**
- Bool values from module-wrapped builtins return I64 (1/0) not I1 (true/false)
- E2E module tests require top-level `let _ = expr` (no `in` chaining)
