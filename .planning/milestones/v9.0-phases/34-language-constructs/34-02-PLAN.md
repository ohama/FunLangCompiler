**lang_runtime.c** — Add the following function AFTER the existing `lang_for_in_list` or `lang_for_in_array` function. This function is similar to `lang_for_in_list` but instead of discarding the closure return value, it accumulates results into a new `LangCons*` list:

```c
LangCons* lang_list_comp(void* closure, void* collection) {
    LangClosureFn fn = *(LangClosureFn*)closure;
    LangCons* cur = (LangCons*)collection;
    // Build result list (reverse accumulate, then reverse)
    LangCons* result = NULL;
    while (cur != NULL) {
        int64_t val = fn(closure, cur->head);
        LangCons* cell = (LangCons*)GC_malloc(sizeof(LangCons));
        cell->head = val;
        cell->tail = result;
        result = cell;
        cur = cur->tail;
    }
    // Reverse the accumulated list to preserve order
    LangCons* reversed = NULL;
    while (result != NULL) {
        LangCons* next = result->tail;
        result->tail = reversed;
        reversed = result;
        result = next;
    }
    return reversed;
}
```

**Key insight:** The Range node `0..n` elaborates to `@lang_range` which returns a `LangCons*` list. So the range form `[for i in 0..n -> expr]` naturally works — `lang_list_comp` receives a `LangCons*` list from `@lang_range` and iterates it.
