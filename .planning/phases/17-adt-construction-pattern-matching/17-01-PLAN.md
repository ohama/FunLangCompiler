<task type="auto">
  <name>Task 1: Add Constructor cases to elaborateExpr and freeVars</name>
  <files>src/LangBackend.Compiler/Elaboration.fs</files>
  <action>
  Add two new cases to `elaborateExpr` (before the catch-all `| _ ->` at line ~1240):

  **Case 1: Nullary constructor** `| Constructor(name, None, _) ->`
  - Look up tag from TypeEnv: `let info = Map.find name env.TypeEnv`
  - GC_malloc(16) for the ADT block
  - GEPLinearOp field 0, store tag (int64 info.Tag) as I64
  - GEPLinearOp field 1, store LlvmNullOp (null Ptr)
  - Return (blockPtr, ops)
  - Follow the exact cons-cell allocation pattern at lines 852-865

  **Case 2: Unary/multi-arg constructor** `| Constructor(name, Some argExpr, _) ->`
  - Look up tag from TypeEnv: `let info = Map.find name env.TypeEnv`
  - Elaborate argExpr: `let (argVal, argOps) = elaborateExpr env argExpr`
  - GC_malloc(16) for the ADT block
  - GEPLinearOp field 0, store tag (int64 info.Tag) as I64
  - GEPLinearOp field 1, store argVal directly (whatever type — I64 for int, Ptr for tuple)
  - Return (blockPtr, argOps @ allocOps)
  - This handles BOTH unary (Some 42) and multi-arg (Pair(3,4)) because multi-arg arg is a Tuple expression that elaborates to Ptr

  **Also add Constructor to freeVars** (before the catch-all `| _ -> Set.empty` at line 121):
  ```fsharp
  | Constructor(_, None, _) -> Set.empty
  | Constructor(_, Some argExpr, _) -> freeVars boundVars argExpr
  ```

  **Also handle App(Var(name), arg) where name is in TypeEnv** — the parser may produce constructor calls as function application. In elaborateExpr, in the existing `App` case (or before the catch-all), add a check: if `App(Var(name, _), arg, _)` and `Map.containsKey name env.TypeEnv`, redirect to `elaborateExpr env (Constructor(name, Some arg, unknownSpan))` or inline the unary constructor logic. Check the parser output by looking at test results — if the parser produces `Constructor` AST nodes directly, this may not be needed.

  **Important:** Use `LlvmGEPLinearOp` (NOT `LlvmGEPStructOp`) for ADT field access. ADT blocks are raw GC_malloc'd memory, not typed structs.
  </action>
  <verify>
  `dotnet build src/LangBackend.Compiler/LangBackend.Compiler.fsproj` compiles without errors.
  Run all 45 existing E2E tests and verify they pass (regression gate).
  </verify>
  <done>elaborateExpr handles Constructor(name, None, _) and Constructor(name, Some argExpr, _); freeVars handles Constructor; build succeeds; all 45 E2E tests pass.</done>
