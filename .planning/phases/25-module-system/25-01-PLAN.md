  <action>
  Three changes in Elaboration.fs:

  **1. Make prePassDecls recursive and add ModuleDecl/NamespaceDecl arms (line ~2317):**
  - Change `let private prePassDecls` to `let rec private prePassDecls`
  - Add two new arms BEFORE the `| _ -> ()` wildcard:
    ```
    | Ast.Decl.ModuleDecl(_, innerDecls, _) ->
        let (iT, iR, iE) = prePassDecls innerDecls
        typeEnv  <- Map.fold (fun acc k v -> Map.add k v acc) typeEnv  iT
        recordEnv <- Map.fold (fun acc k v -> Map.add k v acc) recordEnv iR
        exnTags  <- Map.fold (fun acc k v -> Map.add k v acc) exnTags  iE
    | Ast.Decl.NamespaceDecl(_, innerDecls, _) ->
        let (iT, iR, iE) = prePassDecls innerDecls
        typeEnv  <- Map.fold (fun acc k v -> Map.add k v acc) typeEnv  iT
        recordEnv <- Map.fold (fun acc k v -> Map.add k v acc) recordEnv iR
        exnTags  <- Map.fold (fun acc k v -> Map.add k v acc) exnTags  iE
    ```
  - IMPORTANT: The exnCounter concern from research — each recursive call creates its own counter starting at 0. To fix, pass exnCounter as parameter OR restructure. Simplest: pass `exnCounter: int ref` as a parameter to recursive calls so the tag counter is shared. Change signature to `let rec private prePassDecls (exnCounter: int ref) (decls: Ast.Decl list)` and update the single call site in `elaborateProgram` (line ~2399) to pass `ref 0`.

  **2. Add flattenDecls helper (insert BEFORE extractMainExpr, around line ~2350):**
  ```
  let rec private flattenDecls (decls: Ast.Decl list) : Ast.Decl list =
      decls |> List.collect (fun d ->
          match d with
          | Ast.Decl.ModuleDecl(_, innerDecls, _) -> flattenDecls innerDecls
          | Ast.Decl.NamespaceDecl(_, innerDecls, _) -> flattenDecls innerDecls
          | _ -> [d])
  ```

  **3. Modify extractMainExpr (line ~2355):**
  - Add `let flatDecls = flattenDecls decls` as the first line after `let s = unknownSpan`
  - Change `decls |> List.filter` to `flatDecls |> List.filter`
  - Add `| Ast.Decl.LetPatDecl _ -> true` to the filter (alongside LetDecl, LetRecDecl, LetMutDecl)
  - Add LetPatDecl arms to the `build` function:
    - Single: `| [Ast.Decl.LetPatDecl(pat, body, _)] -> LetPat(pat, body, Number(0, s), s)`
    - With rest: `| Ast.Decl.LetPatDecl(pat, body, _) :: rest -> LetPat(pat, body, build rest, s)`
    Place these BEFORE the `| _ :: rest -> build rest` wildcard arm.

  OpenDecl and NamespaceDecl require NO additional handling in extractMainExpr — they hit the wildcard `| _ -> false` in the filter (or `| _ :: rest -> build rest` in build) and are correctly skipped. flattenDecls already handles NamespaceDecl by recursing into it.
