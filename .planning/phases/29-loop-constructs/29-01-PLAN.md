  ```fsharp
  | WhileExpr (cond, body, _) ->
      Set.union (freeVars boundVars cond) (freeVars boundVars body)
  ```

  This covers LOOP-05 for WhileExpr.

  **2. elaborateExpr case for WhileExpr (add near the end of elaborateExpr, before the final match fallthrough):**

  Use the header-block pattern (3 side blocks) as recommended by research. The pattern is:

  ```
  entry fragment:
    cf.br ^while_header

  ^while_header:
    <elaborate cond>
    cf.cond_br %cond, ^while_body, ^while_exit

  ^while_body:
    <elaborate body, discard value>
    cf.br ^while_header              -- back-edge

  ^while_exit(%unit : i64):
    (empty body -- patched by LetPat continuation)
  ```

  Implementation details:
  - Use `freshLabel env "while_header"`, `freshLabel env "while_body"`, `freshLabel env "while_exit"` for block labels.
  - Elaborate `condExpr` to get `(condVal, condOps)` -- these go in the header block.
  - Elaborate `bodyExpr` to get `(_bodyVal, bodyOps)` -- discard the value. These go in the body block.
  - Create `unitConst` via `ArithConstantOp(unitConst, 0L)` -- place this in the ENTRY fragment (before the `CfBrOp` to header), not in any side block. This ensures it dominates the header where it's used as an arg to `CfCondBrOp`'s false branch.
  - Create `exitArg` as the block argument of `^while_exit` with type I64.
  - The header block's `CfCondBrOp` passes `[unitConst]` as falseArgs to `^while_exit`.
  - The body block ends with `CfBrOp(headerLabel, [])` -- unconditional back-edge to header (no args since header has `Args = []`).
  - Push 3 blocks to `env.Blocks.Value`: header, body, exit (exit with `Body = []`).
  - Return `(exitArg, [ArithConstantOp(unitConst, 0L); CfBrOp(headerLabel, [])])` as the entry fragment.

  IMPORTANT notes:
  - Do NOT elaborate condExpr twice. The header-block pattern means condition is only elaborated once (inside the header block). The body block branches back to the header which re-executes the condition ops.
  - The body block must handle the case where bodyExpr itself produces side blocks (e.g., body contains an if-else). The bodyOps go into the body block's Body, and any side blocks from elaborating bodyExpr are already appended to env.Blocks.Value by the recursive call. Push the while's 3 blocks AFTER elaborating both condExpr and bodyExpr so the while blocks come after any inner blocks.
  - The exit block must have `Body = []` (empty) so LetPat/Let patching works correctly.
  - Follow the same `env.Blocks.Value <- env.Blocks.Value @ [...]` pattern used by `If` (line 680-685).
  </action>
