  <name>Task 1: Add expandImports to Program.fs</name>
  <files>src/LangBackend.Cli/Program.fs</files>
  <action>
Add two private functions and integrate them into the `main` function pipeline:

1. **`resolveImportPath`** (private): Takes `importPath: string` and `importingFile: string`. If `importPath` is rooted (`Path.IsPathRooted`), return it as-is. Otherwise, get the directory of the importing file via `Path.GetDirectoryName(Path.GetFullPath(importingFile))`, then `Path.GetFullPath(Path.Combine(dir, importPath))`.

2. **`expandImports`** (private, recursive): Takes `visitedFiles: HashSet<string>`, `currentFile: string`, `decls: Ast.Decl list`. Returns `Ast.Decl list`. Uses `List.collect` over decls:
   - `FileImportDecl(importPath, span)`: Resolve path. If `visitedFiles.Contains resolvedPath`, `failwithf "Circular import detected: %s is already being imported" resolvedPath`. If file not found, `failwithf "Import not found: %s (resolved to %s from %s)" importPath resolvedPath currentFile`. Add to visitedFiles, try: read file, `parseProgram src resolvedPath` (NO prelude prefix), extract decls from Module variant, recursively `expandImports`, finally: remove from visitedFiles. Return expanded decls.
   - `ModuleDecl(name, innerDecls, s)`: Return `[Ast.Decl.ModuleDecl(name, expandImports visitedFiles currentFile innerDecls, s)]`
   - `NamespaceDecl(name, innerDecls, s)`: Same as ModuleDecl — recurse into inner decls.
   - Any other decl: Return `[other]`

   Handle all Module variants when extracting decls: `Module(ds, _)` / `NamedModule(_, ds, _)` / `NamespacedModule(_, ds, _)` -> ds, `EmptyModule _` -> [].

3. **Integration in `main`**: After line `let ast = parseProgram combinedSrc inputPath`, add import expansion:
   ```
   let expandedAst =
       match ast with
       | Ast.Module(ds, s) ->
           let visited = System.Collections.Generic.HashSet<string>()
           visited.Add(System.IO.Path.GetFullPath(inputPath)) |> ignore
           Ast.Module(expandImports visited inputPath ds, s)
       | Ast.NamedModule(nm, ds, s) ->
           let visited = System.Collections.Generic.HashSet<string>()
           visited.Add(System.IO.Path.GetFullPath(inputPath)) |> ignore
           Ast.NamedModule(nm, expandImports visited inputPath ds, s)
       | Ast.NamespacedModule(nm, ds, s) ->
           let visited = System.Collections.Generic.HashSet<string>()
           visited.Add(System.IO.Path.GetFullPath(inputPath)) |> ignore
           Ast.NamespacedModule(nm, expandImports visited inputPath ds, s)
       | other -> other
   ```
   Then pass `expandedAst` (not `ast`) to `Elaboration.elaborateProgram`.

IMPORTANT:
- Do NOT modify Elaboration.fs. Expansion happens entirely in Program.fs.
- Imported files are parsed with `parseProgram src resolvedPath` — no prelude prefix. Prelude bindings are already in the outer `combinedSrc`.
- Use `HashSet` push/pop (Add in try, Remove in finally) — NOT a global visited set. Diamond imports are NOT cycles.
- Place the two helper functions between `parseProgram` and `main` in the file.
