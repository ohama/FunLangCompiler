```bash
echo "x" > /tmp/test_unbound.fun

```

Use the actual output to write the expected output in the .flt file. The format should be something like:
`/tmp/test_unbound.fun:1:1: Elaboration: unbound variable 'x'`

But since the test uses `%S` substitution for the directory, the .flt expected output must use the actual path that the test runner will produce. Look at existing error tests (like 06-02-cli-error.flt) to understand the exact format.
