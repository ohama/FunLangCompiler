Fix Issue #22: `s.[i]` returns wrong value when `s` is a function parameter.

Output: Patched ElabHelpers.fs and Elaboration.fs, plus an E2E test proving the fix.

**Bug A fix — ElabHelpers.fs `hasParamPtrUse` (around line 561):**

Add an `IndexGet` case inside `hasParamPtrUse` in `isPtrParamBody`. Insert it before the "Recurse through let bindings" comment (before line 579). The new case:

```fsharp
// Phase 94: IndexGet on param → param is Ptr (string/array/hashtable all need Ptr)
| IndexGet(Var(v, _), _, _) when v = paramName -> true
```

This makes `isPtrParamBody` correctly return `true` when the function body does `s.[i]` on the parameter, so the parameter is typed as `!llvm.ptr` in the MLIR signature.

**Bug B fix — Elaboration.fs single-param Let-Lambda (around line 340):**

The current match is:
```fsharp
| Let (name, StripAnnot (Lambda (param, body, lamSpan)), inExpr, _)
```

This uses `StripAnnot` which erases `LambdaAnnot` info. To detect explicit string type annotations, we need access to the original bind expression.

Restructure the match to preserve the original expression. Change the match pattern to:
```fsharp
| Let (name, bindExprOrig, inExpr, _)
    when (match stripAnnot bindExprOrig with
          | Lambda (param, body, _) ->
              freeVars (Set.singleton param) body
              |> Set.filter (fun v -> Map.containsKey v env.Vars)
              |> Set.isEmpty
          | _ -> false) ->
    let (Lambda (param, body, lamSpan)) = stripAnnot bindExprOrig
```

Then after computing `paramType` (line 344), add:
```fsharp
let paramIsString =
    match Map.tryFind lamSpan env.AnnotationMap with
    | Some (Type.TArrow(Type.TString, _)) -> true
    | _ ->
    match bindExprOrig with
    | Ast.LambdaAnnot(p, Ast.TEName "string", _, _) when p = param -> true
    | _ -> false
```

Then change the `StringVars = Set.empty` in bodyEnv (line 368) to:
```fsharp
StringVars = if paramIsString then Set.singleton param else Set.empty
```

**Bug B fix (LetRec) — Elaboration.fs LetRec case (around line 772):**

The LetRec binding tuple already has `_paramTypeAnnot` available. In the `List.map` at line 772, change:
```fsharp
bindings |> List.map (fun (name, param, _paramTypeAnnot, body, bindingSpan) ->
```
to use the paramTypeAnnot:
```fsharp
bindings |> List.map (fun (name, param, paramTypeAnnot, body, bindingSpan) ->
```

After computing `paramType` (line 773), add:

Update the binding tuple result (line 784) to include `paramIsString`:

Then in the `List.fold` at line 793, update the destructuring to include `paramIsString`:

And change `StringVars = Set.empty` in bodyEnv (line 805) to:
