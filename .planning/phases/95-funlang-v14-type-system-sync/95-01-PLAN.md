**Edit 1 — `typeNeedsPtr` (line ~621):**

Change the first arm of the match from:
```fsharp
| Type.TString | Type.TList _ | Type.TArray _ | Type.THashtable _
| Type.TArrow _ | Type.TExn -> true
```
to:
```fsharp
| Type.TString | Type.TList _ | Type.TArray _ | Type.THashtable _
| Type.THashSet _ | Type.TQueue _ | Type.TMutableList _ | Type.TStringBuilder
| Type.TArrow _ | Type.TExn -> true
```

All four new types are heap-allocated and need Ptr representation.

**Edit 2 — `detectCollectionKind` (line ~132):**

The match on `inferredType` already has arms for `TData("HashSet",_)` etc. which were the old workaround via TData. Now add dedicated type arms BEFORE those TData arms so the new proper union cases are matched first:

Change lines 132-137 from:
```fsharp
    match inferredType annotationMap expr with
    | Some (Type.TData("HashSet", _)) -> Some HashSet
    | Some (Type.TData("Queue", _)) -> Some Queue
    | Some (Type.TData("MutableList", _)) -> Some MutableList
    | Some (Type.THashtable _) -> Some Hashtable
    | Some _ -> None
```
to:
```fsharp
    match inferredType annotationMap expr with
    | Some (Type.THashSet _) -> Some HashSet
    | Some (Type.TQueue _) -> Some Queue
    | Some (Type.TMutableList _) -> Some MutableList
    | Some (Type.TData("HashSet", _)) -> Some HashSet
    | Some (Type.TData("Queue", _)) -> Some Queue
    | Some (Type.TData("MutableList", _)) -> Some MutableList
    | Some (Type.THashtable _) -> Some Hashtable
    | Some _ -> None
```

Keep the old TData arms as fallback for backward compatibility with any pre-v14 typed ASTs.
