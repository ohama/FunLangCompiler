## 7. Tech Debt / Observations

1. **`lang_index_get_str`/`lang_index_set_str` wrappers retained** -- These exist solely for `ht.["key"]` syntax where the compiler dispatches on Ptr type. Could be eliminated by having the compiler PtrToInt the string key and call `lang_index_get`/`lang_index_set` directly. Low priority.

2. **HashSet stores raw untagged ints** -- This required a separate `lang_hs_hash` function. If HashSet were changed to store tagged values, it could share `lang_ht_hash`. Not blocking but worth noting for future consistency.

3. **Mixed-key hashtables possible** -- The unified API allows storing both int and string keys in the same hashtable. This is correct behavior per the LSB dispatch design, but was previously prevented by separate types. Not a bug, potentially a feature.
